h1 => [ Drupal Interview Questions ]
p => [  A list of top frequently asked  Drupal interview questions  and answers are given below.  ]
h3 => [ 1) Explain Drupal. ]
p => [ Drupal is free, open-source content management framework written in PHP. Drupal is distributed under GNU General Public License.  ]
p => [ Drupal is a framework which organize, manage and publish content easily and provides a lot of features of customization.  ]
h3 => [ 2) What are some most commonly used PHP based CMS? ]
p => [ Following are some most commonly used PHP based CMS: ]
li => [ Drupal ]
li => [ Joomla ]
li => [ WordPress ]
li => [ TYPO3 ]
h3 => [ 3) Why Drupal is so popular and called a powerful framework?  ]
p => [ Drupal is very easy to learn and handle. A user with nil programming skill can also handle the content management and customization of a Drupal's website. That's why Drupal is getting popular day by day.  ]
h3 => [ 4) What are system requirements for Drupal installation? ]
p => [ System requirements for Drupal 8: ]
li => [ Web server: Apache, Nginx, or Microsoft IIS  ]
li => [ Database: MySQL 5.5.3/MariaDB 5.5.20/Percona Server 5.5.8 or higher with PDO and an InnoDB-compatible primary storage engine, PostgreSQL 9.1.2 or higher with PDO, SQLite 3.6.8 or higher ]
li => [ PHP 5.5.9 or higher ]
li => [ PHP Memory: 60 MB  ]
h3 => [ 5) Explain Drupal's features. ]
p => [ Drupal has a lot of features: ]
li => [ Simple and secure theme ]
li => [ Mobile first ]
li => [ Provides multilingual features ]
li => [ Easy configuration  ]
li => [ Easy writing  ]
li => [ Quick edits ]
li => [ Better support ]
li => [ Built-in web services ]
li => [ Fast loading speed  ]
h3 => [ 6) What is module in Drupal? ]
p => [ A module is a set of codes that extend Drupal features and functionality.  ]
strong => [ Drupal modules:  ]
p => [  Core modules:  These modules are included with the main download of Drupal, and you can turn on their functionality without installing additional software. ]
p => [  Contributed modules:  These modules are downloaded from the Modules download section of drupal.org, and installed within your Drupal installation. ]
p => [ You can also create your own module "CustomModules" using PHP programming, and Drupal's module API. ]
h3 => [ 7) Write down some modules that are used in Drupal? ]
p => [ Some of the modules recommended are: ]
li => [ Views ]
li => [ Token ]
li => [ Ctools ]
li => [ Quicktabs ]
li => [ Pathauto ]
h3 => [ 8) What is a sub theme in Drupal? ]
p => [ Sub themes are just like other themes of Drupal but they have a little difference. Sub themes inherit the parent's theme. There are many sub themes that can be connected to their parents. A sub-theme can be a child of another sub-theme, and it can be branched and organized however you see fit. This is what gives sub-themes great potential. ]
h3 => [ 9) What does PDO mean in Drupal? ]
p => [ PDO stands for PHP Data Object. It is a lean and consistent way to access databases. It facilitates developers to write portable codes easily. PDO is not an abstraction layer. It is more like a data access layer which uses a unified API.  ]
h3 => [ 10) What is DRUSH in Drupal? ]
p => [ DRUSH is a command line shell and UNIX scripting interface for Drupal.  ]
h3 => [ 11) Can Drupal run on command line? ]
p => [ Yes. Drupal can run on command line by using DRUSH.  ]
h3 => [ 12) Why is Ctools used in Drupal? ]
p => [ Ctools is a set of APIs and tools which is used to improve the developer experience. ]
h3 => [ 13) What are the render arrays in Drupal? ]
p => [ Render arrays are the basic building blocks in Drupal. They provide you a structured way to programmatically change the content before it is displayed.  ]
h3 => [ 14) What are patches in Drupal? ]
p => [ Patches are the files that contain a list of differences between one set of files, and another. You can make changes like addition, deletion in codes through patches. Patches can also be used to make changes to another copy of the original or main file.  ]
h3 => [ 15) Is it possible to disable a core required modules through Drupal admin? ]
p => [ No. It is not possible to disable a core required module through Drupal admin. ]
h3 => [ 16) Explain the database system in Drupal. ]
p => [ In Drupal database system, each type of information has its own database table. Different types of information will be stored in different database table. For example, nodes information is stored in nodes table. ]
h3 => [ 17) What are the supported databases by Drupal? ]
p => [ Drupal supports MySQL/MariaDB, PostgreSQL, and in Drupal 7 SQLite. ]
h3 => [ 18) How can you drop a table using module in Drupal? ]
p => [ To drop a table you can use db_drop_table in install file. ]
h3 => [ 19) What is Drupal distributions and when to use it. ]
p => [ Distributions are full copies of Drupal including Drupal core along with additional software such as modules, libraries and installation profiles. ]
p => [ Distributions are used for: ]
li => [ Demoing Drupal ]
li => [ Evaluating Drupal ]
li => [ Quickly binding Site ]
li => [ Learning Drupal ]
h3 => [ 20) What is Drupal weight? ]
p => [ The term weight in Drupal is used to order nodes. Nodes with lower weight will be at top of the list while nodes with heavier weight will be at lower position. ]
h3 => [ 21) What is Drupal cron? ]
p => [ The term cron is a daemon that executes commands at specified interval. Drupal cron executes commands or scripts automatically at specified interval. ]
h3 => [ 22) Name the translations available in Drupal? ]
p => [ Languages in which Drupal can be translated are: ]
li => [ Catalan ]
li => [ Dutch ]
li => [ French ]
li => [ Hungarian ]
h3 => [ 23) Name the five conceptual layers in Drupal system. ]
li => [ Data (nodes, etc) ]
li => [ Modules ]
li => [ Blocks and menus ]
li => [ User permissions ]
li => [ Themes and templates ]
h3 => [ 24) Explain the API in Drupal 7? ]
p => [ The field API allows custom data fields to be attached to Drupal entities. Any entity type can use field API to make itself "fieldable" means allows fields to attach to it. ]
h3 => [ 25) What is node in Drupal? ]
p => [ Drupal treats all the stored content on its web site as nodes. Any piece of content like blog, article, page, etc is considered as node. Although, the comments are not stored as nodes but they are always connected to a node. ]
h3 => [ 26) What is template.php in Drupal? ]
p => [ The template.php file in the theme directory allows you to override a theme function. ]
h3 => [ 27) Why 'search' function in Drupal is better than other CMS? ]
p => [ Drupal integrates well with external search engines such as Xapian, Apache solr if the built-in search engine does not meet your needs properly. ]
h3 => [ 28) Explain hooks in Drupal. ]
p => [ Hooks are Drupal API code which allows module to interact with Drupal core. Its module system is based on the hooks concept. ]
h3 => [ 29) How to interact with Drupal search systems? ]
p => [ There are three ways to interact with Drupal search system by implementing: ]
li => [ nodeapi (update index) ]
li => [ hook_search() ]
li => [ hook_update_index() ]
h3 => [ 30) How will you add custom PHP codes in Drupal pages or blocks? ]
p => [ Be default, Drupal doesn't allow the insertion of PHP codes in its pages. To embed PHP codes, you need to activate a Drupal module called PHP filter. By default, this module remains disabled. ]
h3 => [ 31) Explain the use of Sections module in Drupal. ]
p => [ The Sections module allows applying different themes to different pages in a Drupal site. ]
h3 => [ 32) Explain Taxonomy in Drupal. ]
p => [ Drupal classifies its content which is called taxonomy and is implemented in the core Taxonomy module. ]
