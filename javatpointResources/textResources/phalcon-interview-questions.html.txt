h3 => [ 1) What is Phalcon? ]
p => [ Phalcon is an open-source framework of PHP programming language. Phalcon is the first framework that implements ORM in C-programming language. ]
h3 => [ 2) Enlist the features of Phalcon. ]
li => [ Low overhead ]
li => [ MVC &amp; HMVC Pattern ]
li => [ Dependency Injection ]
li => [ Support for Rest ]
li => [ Autoloader ]
li => [ Router ]
h3 => [ 3) What is loader? ]
p => [  Loader is a class found under  Phalcon\Loader  directory. It also handles error such as if a class does not exist but it is called in any part of program then special handler is called for handling. ]
h3 => [ 4) What is lazy initialization? ]
p => [  In  Loader  if a class is added according to its need in program, performance is increased as only specific file is included. This technique is known as  lazy initialization . ]
h3 => [ 5) Enlist the methods of loader. ]
li => [ Registering Namespaces ]
li => [ Registering Directories ]
li => [ Registering Classes ]
li => [ Registering Files ]
h3 => [ 6) How can we access to backend in Phalcon? ]
p => [ It is provided by logging services for application. We can login to different backend using different adapters. It offers transaction logging, configuration options, different formats and filters. ]
h3 => [ 7) Enlist the database engines supported by Phalcon. ]
li => [ Phalcon\Db\Adapter\Pdo\Mysql : Mysql ]
li => [ Phalcon\Db\Adapter\Pdo\Postgresql : Postgresql ]
li => [ Phalcon\Db\Adapter\Pdo\Sqlite : SQLite ]
h3 => [ 8) What is PHQL? ]
p => [ PHQL (Phalcon Query Language) allows user to implement query language similar to SQL query language. PHQL is implemented as a parser which connects to RDBMS. ]
h3 => [ 9) Enlist the features of PHQL. ]
li => [ It secures the code using bound parameters. ]
li => [ It prevents injection by executing one SQL statement per call. ]
li => [ It ignores all comments which mostly used in SQL injections. ]
li => [ It only allows data manipulation statement to execute. ]
h3 => [ 10) What is Phalcon models? ]
p => [ Model consists of information or data of the application. It manipulates the data by managing the rules. ]
h3 => [ 11) Enlist the following services provided by Phalcon. ]
li => [ Database Independence ]
li => [ CRUD functionality ]
li => [ Advanced finding capabilities ]
li => [ Ability to relate models ]
h3 => [ 12) What is zephir in Phalcon? ]
p => [ Zephir stand for Ze(nd Engine) Ph(p) I(nt)r(mediate). It is used for creation and maintainability of extensions for PHP. ]
h3 => [ 13) What are cookies and its type? ]
p => [ Cookies stores small text files in browser. It is known as browser cookies. ]
p => [ Types of cookies are: ]
li => [ Session Cookies
 ]
li => [ Persistent Cookies
 ]
h3 => [ 14) Explain routing in Phalcon. ]
p => [ Routing managed by its Router component. Router components allow us to define routes that are mapped to controllers or handlers that receives the request from the application. ]
h3 => [ 15) What is Session? ]
p => [ Sessions are server-side information storage which helps in user interaction with the website or web application. Each session is consists of unique session ID, which is passed to the web server whenever the browser makes an HTTP request. ]
h3 => [ 16) What is CSRF? ]
p => [ CSRF stands for Cross Site Request Forgery. CSRF is created to prevent the form values from being sent outside our application. It generates a random nonce (token) in each form. ]
h3 => [ 17) How can we increase CSRF timeout in Phalcon ? ]
p => [ CSRF timeout is increased by increasing the token time because tokens default uses sessions. ]
h3 => [ 18) Which protocol is used to encrypt the password in Phalcon? ]
p => [ In Phalcon, md5, base64 and sh1 protocol is used to encrypt the password. ]
h3 => [ 19) Which template engine is used in Phalcon? ]
p => [ Phalcon uses Volt templating Engine. Volt provides Phalcon fast execution as it is very fast and designer friendly templating language written in C for PHP. Volt is inspired by Jinja and written by Armin Ronacher. ]
h3 => [ 20) Explain ODM in Phalcon. ]
p => [ ODM stands for Object Document Mapper. It offers a CRUD functionality, events and validations among other services in Phalcon. ]
