h3 => [ 3) What is session bean? ]
p => [ Session Bean encapsulates business logic. It can be invoked by local, remote or web service client. ]
p => [ There are 3 types of session bean. ]
li => [ Stateless Session Bean ]
li => [ Stateful Session Bean ]
li => [ Singleton Session Bean ]
a => [ More details... ]
h3 => [ 4) What is stateless session bean? ]
p => [  A Stateless session bean is a business object that doesn't maintain the conversational state with the client.  More details... ]
h3 => [ 5) Write down the steps for the creation of stateless EJB. ]
li => [ Create a local interface. ]
li => [ The interface is to be used by the client application. ]
li => [ In case the EJB client environment is the same, use @Local annotation. ]
li => [ In case the EJB client environment is different, use @Remote annotation. ]
li => [ Create a stateful session bean. ]
li => [ To signify a stateful bean, use @Stateful annotation. ]
h3 => [ 6) What is stateful session bean? ]
p => [  A Stateful session bean is a business object that maintains the conversational state with the client.  More details... ]
h3 => [ 7) What is singleton session bean? ]
p => [ Singleton session bean is instantiated only once for the application. It exists for the life cycle of the application. ]
h3 => [ 8) What is JMS? ]
p => [  Java Message Service is a messaging service to create, send and receive messages asynchronously.  More details... ]

h3 => [ 10) What is PTP model? ]
p => [  In Point to Point model, one message is delivered to one receiver only. Here, Queue is used as a message-oriented middleware.  More details... ]
h3 => [ 11) What is Pub/Sub model? ]
p => [  In the Publisher/Subscriber model, one message is delivered to all subscribers. Here, Topic is used as a message-oriented middleware.  More details... ]
h3 => [ 12) What is MDB? ]
p => [  Message Driven Bean (MDB) encapsulates business logic. It is invoked by passing the message. It is like JMS receiver.  More details... ]
h3 => [ 13) What is Entity Bean? ]
p => [  Entity Bean is a server-side component that represents the persistent data. Since EJB 3.x, it is replaced by JPA.  More details... ]
h3 => [ 14) What is Session Facade? ]
p => [ Session Facade is a design pattern to access enterprise bean through the local interface. It abstracts the business object interactions and provides a service layer. It makes the performance fast over the network. ]
