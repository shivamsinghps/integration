h1 => [  Jenkins Interview Questions ]
p => [  A list of top frequently asked  Jenkins  interview questions and answers are given below. ]
h3 => [ 1) What is Jenkins? ]
p => [ Jenkins is an open source continuous integration tool written in Java. It keeps a track on version control system and to initiate and monitor a build system if changes occur. ]
h3 => [ 2) What is the difference between Maven, Ant and Jenkins? ]
p => [ The most basic difference is: ]
p => [ Maven and Ant are Build Technologies whereas Jenkins is a continuous integration tool.  ]
h3 => [ 3) Which SCM tools does Jenkins support? ]
p => [ Jenkins supports the following SCM tools: ]
li => [ AccuRev ]
li => [ CVS ]
li => [ Subversion ]
li => [ Git ]
li => [ Mercurial ]
li => [ Perforce ]
li => [ Clearcase ]
li => [ RTC ]
h3 => [ 4) What is continuous integration in Jenkins? ]
p => [ In software development, multiple developers or teams work on different segments of same web application so you have to perform integration test by integrating all modules. In order to do that an automated process for each piece of code is performed on daily bases so that all your codes get tested. This process is known as continuous integration. ]
h3 => [ 5) What is the relation between Hudson and Jenkins? ]
p => [ Hudson was the earlier name and version of current Jenkins. After some issue, the project name was changed from Hudson to Jenkins. ]
h3 => [ 6) What is the requirement for using Jenkins? ]
p => [ For using Jenkins, you have to need a source code repository which is accessible. For example, a Git repository and a working build script, e.g., a Maven script, checked into the repository. ]
h3 => [ 7) What are the advantages of Jenkins? ]
p => [  Advantage of Jenkins includes: ]
li => [ Bugs tracking are easy at early stage in development environment. ]
li => [ Provides a large numbers of plugin support. ]
li => [ Iterative improvement to the code. ]
li => [ Build failures are cached at integration stage. ]
li => [ For each code commit changes an automatic build report notification generates. ]
li => [ To notify developers about build report success or failure, it is integrated with LDAP mail server. ]
li => [ Achieves continuous integration agile development and test driven development. ]
li => [ With simple steps, maven release project is automated. ]
h3 => [ 8) How to make sure that your project builds doesn?t break in Jenkins? ]
p => [ You must follow these steps to make sure that your project builds doesn?t break in Jenkins: ]
li => [ First, perform successful clean install on your local machine with all unit tests. ]
li => [ Check all your code changes. ]
li => [ Synchronize with repository to make sure that all required config and POM changes and any difference is checked into the repository. ]
h3 => [ 9) How can you move or copy Jenkins from one server to another? ]
p => [  Follow these steps to move or copy Jenkins from one server to another: ]
li => [ First, copy the related job directory and slide a job from one installation of Jenkins to another. ]
li => [ Make a copy of an already existing job by making clone of a job directory by a different name. ]
li => [ Renaming an existing job by rename a directory. ]
h3 => [ 10) Which commands can be used to start Jenkins manually? ]
p => [ You can use any one of the following commands to start Jenkins manually: ]
li => [ (Jenkins_url)/restart: Forces a restart without waiting for builds to complete. ]
li => [ (Jenkin_url)/safeRestart: Allows all running builds to complete. ]
h3 => [ 11) What are the most useful plugins in Jenkins? ]
p => [  Some most useful plugins in Jenkins: ]
li => [ Maven 2 project ]
li => [ Amazon EC2 ]
li => [ HTML publisher ]
li => [ Copy artifact ]
li => [ Join ]
li => [ Green Balls ]
h3 => [ 12) How to create a backup and copy files in Jenkins? ]
p => [ If you want to create a back-up of your Jenkins setup, just copy the directory that saves all the setting, build artifacts and logs of Jenkins in its home directory. You can also copy a job directory to clone or replicate a job or rename the directory. ]
h3 => [ 13) How can you clone a Git repository via Jenkins? ]
p => [ If you want to clone a Git repository via Jenkins, you have to enter the e-mail and user name for your Jenkins system. Switch into your job directory and execute the "git config" command for that. ]
h3 => [ 14) How can you setup Jenkins jobs? ]
p => [ Follow these steps: ]
li => [ Select new item from the menu. ]
li => [ After that enter a name for the job and select free-style job. ]
li => [ Then click OK to create new job in Jenkins. ]
li => [ The next page enables you to configure your job. ]
h3 => [ 15) What are the two components Jenkins is mainly integrated with? ]
p => [ Jenkins is integrated with these two components: ]
li => [ Version Control system like GIT,SVN ]
li => [ And build tools like Apache Maven. ]
