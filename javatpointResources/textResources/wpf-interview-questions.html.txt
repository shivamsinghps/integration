h1 => [ WPF Interview Questions ]
p => [ There is given frequently asked WPF interview questions and answers that has been asked in many companies. Let's see the list of top WPF interview questions. ]
h3 => [ 1) What is WPF? ]
p => [ WPF stands for Window Presentation Framework. It is the latest presentation API by Microsoft Windows. WPF is a 2D and 3D graphics engine.  ]
h3 => [ 2) What are the capabilities of WPF? ]
p => [ WPF has following capabilities: ]
li => [ It has all the equivalent common user controls like buttons, checkboxes, sliders etc.  ]
li => [ It has all the capabilities of HTML and Flash. ]
li => [ It supports fix and flow format document. ]
li => [ It provides the facility of data binding, animation and multimedia.  ]
h3 => [ 3) What are resources in WPF? ]
p => [ In WPF, resources are used to provide a simple way to reuse commonly defined objects and values. They also facilitate you to set the properties of multiple controls at a time. For example, you can set the background property on several elements in a WPF application using a single resource. ]
h3 => [ 4) What is XAML? ]
p => [ XAML is a declarative XML based language. It facilitates you to define objects and properties in XML. Its documents are loaded by XAML parser.  ]
h3 => [ 5) What is the use of XAML? ]
p => [ XAML is used to describe the objects, properties and their relation in between them. It makes you able to create any type of objects i.e. graphical and non-graphical.  ]
h3 => [ 6) What is a routed event? ]
p => [ There are many types of elements in a typical WPF application and these elements are set in an element tree relationship. A routed event is a type of event that is used to invoke handlers on multiple listeners in an element tree.  ]
h3 => [ 7) How many types of documents are supported by WPF? ]
p => [ Two types of documents are supported by WPF. ]
p => [  Flow format document : Flow format document is used to alter the content to fit on the screen size. ]
p => [  Fixed format document : Fixed format document present content irrespective of the screen size.  ]
h3 => [ 8) What do you know by content alignment in WPF? ]
p => [ There are two types of content alignment in WPF: ]
li => [ HorizontalContentAlignment ]
li => [ VerticalContentAlignment ]
p => [  These both properties are defined in the  System.Windows.Controls.Control  class which is the parent class of all controls in WPF. ]
p => [ By default the vertical and horizontal alignment of content of a TextBox is left and top and button is center. ]

p => [  elementName:  Name of the element that uses the resource. ]
p => [  propertyName:  Name of the property that takes its value from the resource. ]

li => [ Static Resource ]
li => [ Dynamic Resource ]
h3 => [ 11) Which namespace is required for working with 3D? ]
p => [ System.Windows.Media.Medi3D is the namespace required for working with 3D. ]
h3 => [ 12) Can we say that WPF is the replacement of DirectX? ]
p => [ No, WPF cannot replace DirectX, because WPF is not enough efficient to create games with stunning graphics. WPF is only meant to be a replacement of windows forms, not DirectX.  ]
h3 => [ 13) What are the dependency properties? ]
p => [ Dependency properties are the type of properties which belong to a specific class but can be used for another class.  ]
h3 => [ 14) What is CLR? ]
p => [ CLR stands for Common Language Runtime. It is a run time environment for .NET ]
h3 => [ 15) What are the freezable objects in WPF? ]
p => [ An object which is unchangeable is known as freezable object. The freezable objects perform better and also safer if they are required to be shared between threads.  ]
h3 => [ 16) What is value converter in WPF? ]
p => [ A value converter acts as a bridge between a target and a source and it is necessary when a target is bound with one source, for instance you have a text box and a button control. You want to enable or disable the button control when the text of the text box is filled or null. In this case you need to convert the string data to Boolean. This is possible using a Value Converter. To implement Value Converters, there is the requirement to inherit from I Value Converter in the System.Windows.Data namespace and implement the two methods Convert and Convert Back. ]
h3 => [ 17) What is the difference between Silverlight and WPF browser application? ]
p => [ These are the main differences between the Silverlight and WPF browser application: ]
li => [ .NET framework is needed for running WPF browser application on the client machine while Silverlight runs using only plug-in.  ]
li => [ The applications made in WPF depend on the operating system because .NET framework only runs on Windows. On the other hand, Silverlight plug-ins can be installed on those OS also, which are not Windows. ]
h3 => [ 18) Which tool is required to sketch a mock of your WPF application? ]

li => [ StackPanel ]
h3 => [ 20) What is PRISM? ]
p => [ PRISM is a framework that is used for creating complex applications for WPF, Silverlight or Window phones. It uses MVVM, IC, Command Pattern, DI and separation of concerns to get loose coupling.  ]
h3 => [ 21) What is CustomControl? ]
p => [ CustomControl is used to expand the functions of existing controls. It contains a default style in theme and code file.  ]
h3 => [ 22) What is the use of custom control? ]
p => [ Custom control is the best way to make a control library. It can also be styled or template. ]
h3 => [ 23) What is the Path animation in WPF? ]
p => [ Path animation is a different type of animation. In this animation, the animated objects follow a path set by the path geometry.  ]
h3 => [ 24) What is the meaning of BAML in WPF? ]

h3 => [ 25) What is the difference between XAML and BAML? ]
p => [ The only difference between XAML and BAML is that BAML is a compressed declarative language which is loaded and parsed quicker than XAML.  ]

p => [ Serialization is a process of converting the state of an object to stream of bytes. ]
h3 => [ 31) What is MVVM? ]
p => [ MVVM is stand for Model View ViewModel. It is a framework for making applications in WPF. MVVM is the same as the MVC framework. It is a 3-tier architecture plus one more layer. Loose coupling can be attained by using MVVM.  ]
h3 => [ 32) When did MVVM introduced? ]
p => [ MVVM was introduced by John Gossman in 2005. It was introduced specifically to use with WPF as a concrete application of Martin Fowler's broader Presentation Model pattern. ]
h3 => [ 33) What are the advantages of MVVM? ]
p => [ MVVM has the following advantages: ]
li => [ Modularity ]
li => [ Test driven approach. ]
li => [ Separation UI and Business layer as view and view model. ]
li => [ Code sharing between pages and forms. ]
li => [ Easy to Maintain.  ]
h3 => [ 34) What are the most important features of MVVM? ]
p => [ A list of most important features of MVVM: ]
li => [ MVVM separates the business and presentation layers, like MVP and MVC. ]
li => [ It improves Structure/separation of concerns (View, ViewModel and Model). ]
li => [ It enables a better Design/Developer Workflow. ]
li => [ It improves simplicity and testability. ]
li => [ It is enabled by the robust data binding capability of XAML. ]
li => [ No need to use a code behind file (minimalist code-behind file). ]
li => [ Provides application development ability for multiple environments. ]
li => [ Powerful Data Binding, command, validation and much more. ]
li => [ The designer and developer can work together. ]
h3 => [ 35) How is MVVM different from MVC? ]
p => [ MVC stands for Model-View Controller and.MVVM stands for Model-View ViewModel. ]
p => [ In MVVM, View Model is used instead of a controller. This View Model is present beneath the UI layer. It reveals the command objects and data that the view requires. It acts like a container object from which view gets its actions and data. ]
