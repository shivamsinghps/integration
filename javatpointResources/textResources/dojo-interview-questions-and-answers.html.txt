h1 => [ Dojo Interview Questions ]
p => [  A list of top frequently asked  Dojo interview questions  and answers are given below.  ]
h3 => [ 1) What is the Dojo? ]
p => [ A Dojo is an open source JavaScript library. It provides an easy way to design and develop JavaScript/AJAX-based applications and websites. It is just like a JavaScript toolkit that develops cross-platform applications rapidly.  ]
h3 => [ 2) What are the most prominent features of Dojo? ]
li => [ Dojo is an open source JavaScript toolkit. ]
li => [ It is easy to learn. ]
li => [ It is used to develop highly interactive web applications. ]
li => [ It offers widgets, utilities, and higher IO abstraction. ]
li => [ It is licensed by BSD or AFL. ]
h3 => [ 3) Tell something about the history of Dojo. ]
p => [ Dojo development was first started by Alex Russell and Dylan Schiemann in 2004. The very first Dojo code was written in September 2004. It is released in March 2005.  ]
h3 => [ 4) What is the basic directory structure of Dojo? ]
p => [ The basic directory structure of Dojo is simple and contains the following three points: ]
p => [  /index.html :  The application entry point. ]
p => [  /app:  The application module. ]
p => [  /app/main.js:  The main script for the app module. ]
h3 => [ 5) Explain the Dojo architecture? ]
p => [  Dojo architecture contains the following: ]
li => [ DojoX - It contains various unstable modules. So, still, it is difficult to include it in dojo or dijit.  ]
li => [ Dijit - It is a toolkit of user-interface modules for layout and widgets. ]
li => [ Dojo Core - It includes the core and most non-visual modules. ]
li => [ Dojo Base - It contains the set of functionalities that are available automatically. ]
li => [ Custom code - It includes JavaScript code, HTML markup, and CSS style declarations. ]
li => [ Util - It contains the build tools like optimization, style-checking, documentation, and testing. ]
h3 => [ 6) What are the advantages or benefits of Dojo? ]
p => [ Dojo is a very high-quality JavaScript toolkit. It has several advantages or benefits that support: ]
li => [ Loosely typed variables ]
li => [ Associative arrays ]
li => [ Objects and classes ]
li => [ W3C DOM support in the Dojo ]
li => [ Regular expression ]
li => [ Associative arrays ]
h3 => [ 7) What is the use of DOJO DOM functions? ]
p => [ DOJO DOM functions can be used to create, delete, update, insert and read the elements on the page. Its focus is to improve the cross-browser incompatibilities and makes common operations simple. ]
h3 => [ 8) Mention some of the disadvantages/drawbacks of Dojo? ]
p => [  Disadvantages/ limitation of Dojo: ]
li => [ Dojo supports limited browsers. ]
li => [ You can't hide the Dojo codes in the case of commercial application. ]
li => [ Dojo requires many networks. ]
li => [ Documentation is quite narrow. ]
h3 => [ 9) What is Dijit? ]
p => [ Dijit is a DOJO's user interface library that provides the widget to build web 2.0 GUI's with or without using JavaScript. The Dijit can be used in any of the two ways: ]
li => [ declaratively by using particular attributes inside of regular HTML tags ]
li => [ programmatically through JavaScript ]
h3 => [ 10) Which resources do a Widget consists? ]
p => [ A widget contains the following two resources: ]
li => [ JavaScript file ]
li => [ HTML snippet  ]
h3 => [ 11) What are the features provided by DOJO Widgets? ]
p => [ A widget contains the following interactive features: ]
li => [ Sortable tables ]
li => [ Dynamic charts  ]
li => [ Menus  ]
li => [ Tooltips  ]
li => [ Tabs ]
li => [ 2D vector drawings  ]
h3 => [ 12) What is the use of modules in Dojo? ]
p => [ In DOJO, modules are written in Asynchronous Module Definition format that makes code easier to debug. A single reference can access a module. If you want to expose multiple functions in a module, properties are required on a single object that represents the module. ]
h3 => [ 13) What is the use of Asynchronous module definition (AMD)? ]
p => [ The Asynchronous module definition (AMD) is just a specification for JavaScript. It defines an application programming interface (API) that defines code module along their dependencies and loads them asynchronously if required. ]
h3 => [ 14) What are the advantages of Dojo charting? ]
p => [ These are the main advantages of DOJO charting: ]
li => [ It presents statistical data in a readable manner. ]
li => [ It provides various inbuilt themes and chart types.  ]
li => [ It facilitates to represent data in any way. ]
li => [ It is capable of working on almost all the devices. ]

p => [ A list of components that comes along with Dojo framework: ]
li => [ Base ]
li => [ Dijit ]
li => [ Dojox ]
li => [ Util ]

h3 => [ 18) Which conditions have to be fulfilled for a function of Dojo/ready module record a callback? ]
p => [  The Dojo/ ready module has a function that records a callback that runs once the three conditions have met: ]
li => [ DOM is ready ]
li => [ All outstanding or remaining modules of requested code have completed loading ]
li => [ Other registered functions with a higher priority have completed. ]
h3 => [ 19) What is the difference between Ajax and Dojo? ]
