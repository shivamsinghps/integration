1) What is the WebLogic server, and what are its main usages?

WebLogic Server is an application server designed for Java-based Java EE, which is currently being managed and developed by Oracle Corporation, commonly referred to as Oracle WebLogic Server. Oracle acquired WebLogic Server in 2008 after purchasing BEA Systems. The server is designed to be a comprehensive and customizable platform that enables enterprises to create, deploy, and operate various types of applications, including Java, both on-premises and in the cloud.

The WebLogic server is capable of hosting Java-based applications and offers a reliable, advanced, and scalable implementation of Java Enterprise Edition (EE) as well as Jakarta EE. It facilitates the deployment of web and enterprise applications to execute business logic. In simpler terms, the WebLogic server enables the effortless deployment, distribution, and operation of Java-based applications.

2) What do you understand by deployment descriptors?

Oracle WebLogic Server employs deployment descriptors linked with various modules and applications, presented in XML form. These descriptors primarily outline the contents present within either the directory or jar file. Deployment descriptors, typically following J2EE standards, are instrumental in declaring the standard and the portable descriptors that are compatible with J2EE applications and modules.

6) What is a domain in an Oracle WebLogic server?

Domains are a fundamental aspect of the Oracle WebLogic server, representing a collection of related resources that operate together as a single entity. The resources within a domain are logically grouped and managed as a cohesive unit, providing a structured approach to server administration and maintenance.

A domain is a unit of administration in Oracle WebLogic that comprises an Administration Server instance. This server enables the management and configuration of all resources associated with that domain. Domains can be used to manage diverse applications and organizational divisions, and to logically separate development, testing, and production environments.

8) What do you understand by the Admin Server in Oracle WebLogic Servers?

The Admin Server is an essential component of Oracle WebLogic Servers that functions as a central control point for managing a WebLogic Server domain. Unlike any other servers in the domain, this type of server is called the Admin Server. Its primary role is to create, configure, and delete domain resources such as managed servers, clusters, machines, work managers, data sources, and more. It serves as the central domain configuration interface for managing and organizing domain resources.

When a domain is set up and has only one instance of WebLogic Server, it serves as both the Administration Server and the Managed Server.

9) What is the Managed Server in Oracle WebLogic Servers?

A Managed Server is a specific instance of the WebLogic server that operates on the Java Virtual Machine (JVM) and is configured separately. This server runs various Java components, such as JMS Applications, EJB Applications, Web Applications, and Web Services. It is responsible for executing business applications in a production environment and does not rely on other managed servers in the same domain except for cases where they are not in a cluster. It is possible to have multiple managed servers in a single domain, whose number can range from 0 to N.

In a WebLogic domain, the Administration Server is one type of server instance while the other type is known as the Managed Server. Managed Servers have the ability to start up independently of the Administration Server in case it's unavailable. To improve application scalability and availability, multiple Managed Servers can be configured as a WebLogic Server cluster.

13) What methods can be used to provide user credentials for starting the server?

When a domain is created, the setup wizard prompts the user for important information such as their username and password. These details are required when logging in as the administrator for the first time. If the domain was created in development mode, the setup wizard automatically saves this information in an encrypted format within the identity file. During startup, the system refers to this file to authenticate the user. Should the identity file be unavailable for any reason, the system will prompt the user to enter their credentials. In the event that you need to change your login details or create a domain in production mode, you can create a new identity file.

14) What would happen if the Administration Server fails?

If a domain's Administration Server experiences a failure, it won't impact the operation of Managed Servers in that domain. Managed Servers in such domains will continue to run even if the Administration Server becomes unavailable. For clustered server instances, the domain's load balancing and failover features will still work even in the event of an Administration Server failure. If the Administration Server does go down, Managed Servers in the domain will periodically attempt to reconnect to it.

If there is a malfunction in the hardware or software of the host machine, the entire system can be affected, including the Administration Server and other server instances on the same machine. However, the failure of the Administration Server does not halt the operation of Managed Servers within the domain. Even if the Administration Server is not running, you can still start a Managed Server. The Managed Server will use a local copy of its configuration files to configure itself and will attempt to connect with the Administration Server periodically. If it fails to connect, it will synchronize its configuration state with the Administration Server.

34) What is the key difference between server crash and server hang?

When it comes to server issues, there are two common problems that can occur: server crash and server hang. With a server crash, the Java process completely disappears. However, in the case of a server hang, the Java process becomes unresponsive.
