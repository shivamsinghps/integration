1) Why is Java called platform independent?

Java's platform independence allows it to be executed on different operating systems such as Windows, Mac, or Linux. The reason behind Java's platform independence is due to the bytecode that is generated by the compiler when the code is written. The generated bytecode can be executed in any operating system, provided that the JDK is installed in that system. This unique feature of Java makes it highly portable and compatible with different platforms.

Exception handling plays a crucial role in managing exceptions that arise during program execution. Failure to handle these exceptions can result in program termination. 5) What is OOP concept?

Object-Oriented Programming (OOP) is a coding methodology that revolves around the creation and manipulation of objects and classes. Java, a popular programming language, follows this coding practice. OOP concentrates on concepts such as encapsulation, inheritance, and polymorphism.

6) Explain the basic features of OOPs.

The fundamental characteristics of Object Oriented Programming (OOP) are as follows:

  • Object: - An object is a physical entity which has a state and behaviour. It occupies space in memory. It is a sample of a class. Object helps to access the methods and variables in the program.
  • Class - A Class is "collection of objects." A class is a logical entity, which does not take any space. A class includes all the data and methods which shows the behaviour of an object.
  • Inheritance - Inheritance is a process by which one class can have all properties of other class. Inheritance increases the code reusability. There are two terms used
    • Child class (Subclass): Class which inherits other class, called as Child class or derived class.
    • Parent class (Superclass): A class which got inherited by another class is termed as parent class or Base class.
  • Polymorphism: - Polymorphism is a method of performing "a single task in different ways." Polymorphism enables a programmer to use methods or operators in different ways. In Java, we use method overloading and overriding to obtain the polymorphism.
  • Abstraction: If we show only functionality and hide the explanations or details then this process is called as Abstraction. For achieving the abstraction, we use two ways in Java
    • Abstract class
    • Interface
  • Encapsulation: Encapsulation is a process of enclosing the data and code together to form a single unit. It makes the data safer within the code for any modification. For achieving the encapsulation, we can declare the data variables of class as private.

7) Differentiate between class and object.

Both class and object are essential components of object-oriented programming. However, they have distinct differences. The following outlines the fundamental distinctions between these two concepts.

  • A class is a concept or idea while an object is a tangible thing or entity. The former is a logical representation, while the latter is a physical manifestation.
  • When you create a class in programming, it does not take up any space in memory. However, when you create an object based on that class, it will occupy a certain amount of space in memory.
  • To define a class in Java, the keyword 'class' is used, which is followed by the name of the class. On the other hand, we can create an object in Java by using the 'new' keyword.
  • A Class can be seen as a production line that creates objects, with objects being the various instances of the Class.

11) What is the AVL tree? What is its significance?

The AVL tree was created by Adelson-Velsky and Landis and is named after them. It is a type of self-balancing binary search tree that is height-balanced. This means that the tree is designed to keep the height of its nodes relatively equal as data is added or removed from the tree.

  • The AVL tree has a balancing factor associated with each of its nodes.
  • To ensure balance in a tree structure, it's essential to calculate the balancing factor. This factor represents the allowable difference between the heights of the left and right sub-trees, which must not exceed one for each node in the tree.

The time complexity for insertion in an AVL tree is O(log n) because of its balanced nature. As its height is limited to O(log(n)), the time taken for insertion is also limited to this value.

AVL tree is a self-balanced binary search tree that offers better performance than regular binary search trees. While binary search trees can experience reduced performance when they are unbalanced, AVL trees maintain balance, ensuring a time complexity of O(log(n)). This makes AVL trees an important data structure for applications where fast searching is essential.
