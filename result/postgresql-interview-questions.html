1) What is PostgreSQL?

PostgreSQL is a popular open-source database management system, also known as Postgres or Postgresql. It's widely used for managing large web applications and is known for its strong object-relational capabilities. One of the most powerful features of PostgreSQL is its ability to incorporate four basic concepts that allow users to easily extend the system with additional functionality.

This technology makes use of SQL and enhances its functionalities to assure the secure storage of data. It adheres to the industry's best practices and enables users to retrieve data during request processing.

10) What is Multi-Version Concurrency Control in PostgreSQL? Why is it used?

Multi-Version Concurrency Control or MVCC is a technique used in PostgreSQL to enhance performance in a multi-user environment. It eliminates the need for the database to be locked, which saves time for users. MVCC keeps transactions as records to ensure data consistency. This is different from many other database systems that make use of locks for concurrency control.

11) What is the key difference between multi-version and lock models?

Multi-Version Concurrency Control (MVCC) and lock models differ in how they handle acquiring and releasing locks when multiple transactions are accessing the same data simultaneously. The main benefit of MVCC is that it allows for reading and writing to occur concurrently without contention between acquired locks. This means that reading does not interfere with writing and writing does not block reading, making MVCC a more advantageous approach compared to other lock models.

22) What do you know about the history of PostgreSQL?

The development of the POSTGRES project was initiated by Professor Michael Stonebraker in 1986. The project was funded by various organizations, including the Defense Advanced Research Projects Agency (DARPA), the Army Research Office (ARO), the National Science Foundation (NSF), and ESL, Inc. Over the past three decades, the core platform has undergone active development and is now 100% ACID-compliant, working seamlessly across various operating systems. Additionally, it offers an array of extensions such as the PostGIS database extender.

Michel Stonebraker, an established figure in the database industry, is credited with creating PostgreSQL. His role in its development started back in 1986, where he set out to create a project that would succeed its predecessor, Ingres, which is now owned by Computer Associates. The database was initially named Postgres but has since been renamed PostgreSQL due to its widespread usage of SQL standards in relational databases. Additionally, PostgreSQL is notably recognized as the default database used in MAC OS.

Michael Stonebraker created Postgres in 1986, which was a sequel to the Ingres database system. Postgres was named after Ingres, standing for "after Ingres". Postgres was initially developed from 1986 to 1994 and offered features such as point in time recovery, nested transactions (savepoints), Multi-Version Concurrency Control (MVCC), write-ahead logging for fault tolerance, online/hot backups, tablespaces, and sophisticated query planner/optimizer, and asynchronous replication.

29) What is the difference between clustered index and non clustered index in PostgreSQL?

Clustered and non-clustered indexes are two different types of indexes used in PostgreSQL. A clustered index is used to organize table data rows based on their key values. It can be created using the primary key column to sort table rows. On the other hand, a non-clustered index does not match the physical order of data; instead, it is ordered by the columns that make up the index.

35) What is the difference between PostgreSQL and MongoDB databases?

The distinctive features of PostgreSQL and MongoDB databases can be summarized as follows:

PostgreSQL MongoDB
PostgreSQL is an established database server that follows traditional relational database principles, and it adheres to most SQL standards. In contrast, MongoDB is a NoSQL database system.
PostgreSQL and MongoDB are two different types of databases. PostgreSQL is a traditional SQL-based database system similar to MySQL and Oracle. It is open-source and can be used for free. MongoDB, on the other hand, is a NoSQL database that uses a JSON format. While it has a free version, its enterprise-paid versions are more commonly used.
The source code for PostgreSQL database is primarily written using the C programming language. MongoDB has been developed using the C++ programming language.
PostgreSQL is an RDMS, which stands for Relational Database Management System. MongoDB is a type of database management system that does not follow the traditional relational database model.
PostgreSQL is a database management system that operates on the principles of object-oriented programming. MongoDB is a type of database that is document-oriented in nature.
PostgreSQL can be accessed in several different languages. MongoDB is currently only offered in the English language and no other languages are supported at this time.
According to certain measurements, PostgreSQL has been found to be significantly faster than MongoDB, with speed advantages ranging from 4 to 10 times in some cases. In terms of performance, PostgreSQL is faster than MongoDB. Although, MongoDB is an optimal choice when handling extensive and complex data.

41) What do you understand by WAL or Write-Ahead Logging in PostgreSQL?

In order to maintain the consistency and accuracy of data, PostgreSQL uses a technique called Write-Ahead Logging (WAL). This involves recording all changes and actions in a transaction log, ensuring that data can be recovered in the event of a failure.

The purpose of the Write-Ahead Logging feature is to improve the dependability of the database by recording changes before any updates or modifications are made. This feature maintains the database log in the event of a database failure and guarantees that work can resume from where it was last stopped or interrupted.

42) What is tablespace in PostgreSQL? What is its usage?

43) What are the three phenomena that must be prevented between concurrent transactions in PostgreSQL?

In PostgreSQL, there are four levels of transaction isolation that comply with SQL standards, aimed at preventing three undesirable situations that may arise between concurrent transactions. These situations need to be avoided to ensure the integrity of the data.

  • Dirty read: A transaction is called a dirty read when it reads data written by a concurrent uncommitted transaction.
  • Non-repeatable read: It specifies a transaction that re-reads the data it has previously read and then finds another transaction that has already modified it.
  • Phantom read: It specifies a transaction that re-executes a query, returning a set of rows that satisfy a search condition and then finds that the set of rows satisfying the condition has changed due to another recently-committed transaction.

In order to utilize Oracle, it is necessary to obtain a license. You can use PostgreSQL without any cost, as it is an open-source software that is freely available for usage.
Oracle supports a wide range of server operating systems including, but not limited to, OS X, Linux, Windows, z/OS, AIX, HP-UX, and more. PostgreSQL supports various server operating systems, including Windows, Linux, Unix, Solaris, FreeBSD, NetBSD, and HP-UX.
