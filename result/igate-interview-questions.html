1) Differentiate between low-level language and high-level language?

Since computers cannot comprehend human language, we require a distinct language to interact with them. Computer languages vary, with some being comprehensible to computers and dictating their actions. Therefore, two types of languages exist.

Low-level language and high-level language are two different programming languages used in computer programming. They have distinct differences in terms of complexity and ease of use.

Below are the major distinctions between the two languages:

Low-level language High-level language
Low-level languages are programming languages that are designed for a specific computer architecture, making them machine-dependent. They cannot be executed on different platforms. High-level programming languages are designed to be used on any platform, regardless of whether it's a Mac, Windows, or Linux operating system. These languages are written in a way that does not depend on the specific architecture or hardware of the machine. This makes it easier for developers to create software that can be used on multiple platforms without having to rewrite the code for each one.
Languages that are considered low-level are designed to be easily understandable by machines. High-level programming languages are designed to be easily understandable by humans. Unlike low-level languages which are closer to machine language and more difficult for humans to read and understand, high-level languages use familiar words and syntax that are easier to comprehend.
Creating and assembling a low-level language can be a challenging task. Compared to low-level languages, high-level languages are simpler and faster to write and compile.
Debugging programs written in low-level languages can be a challenging task as it can be hard to identify errors within the code. It is simple to identify a mistake and fix it in a high-level coding language.
Low-level languages are coded in binary machine language, resulting in faster program execution compared to high-level programming languages that use human-readable syntax. Programming languages that are designed to be easily readable and understandable by humans, such as high-level languages, require a conversion process into binary form before they can be executed by a computer. This conversion process can cause high-level languages to be slower compared to lower-level languages.
Machine language is an example of a low-level language. low-level languages

4) What is immutable class and how to create an immutable class in Java?

An immutable class is a type of class that is completely unalterable once an object is created. This means that all values and contents attached to the object cannot be changed. Common examples of immutable classes in Java are the wrapper classes like Integer, Boolean, and String. Additionally, it is possible to create custom immutable classes.

Here are some guidelines to ensure that a class in Java is immutable:

  • One may declare a class as final to prevent it from being inherited by other classes.
  • It is necessary to declare all data members in a class as final to ensure that their values cannot be modified.
  • There is no provision available for setters method to modify the values of specific data fields.
  • It is recommended to provide getter methods for all variables in a program or class. This is because getter methods can help to access and retrieve the values of these variables, which can be useful for performing certain operations or calculations. By including getter methods, the overall functionality and usability of the program or class can be improved.

5) "String class is immutable", explain the reason?

In Java, the String class is considered immutable, which indicates that we cannot modify its value. This is due to the fact that String objects are always cached in a shared pool and can be utilized by different clients. Therefore, modifying a string value for one client could potentially affect all other clients. As a result, every time we attempt to modify a string's value, a new object must be created.

Object-oriented programming has three key features: encapsulation, inheritance, and abstraction. These features allow developers to interact with objects in a structured way.

Encapsulation is a technique that allows us to keep the data members of a class hidden from other classes. This means that the data can only be accessed by the class in which it is defined.

  • Change the access modifier of the class variable to private.
  • One way to assign and retrieve values is by utilizing public Setter and Getter methods. This involves creating methods that can set values to a variable and return values from it, respectively.

Java's inheritance feature allows a class to inherit properties, such as methods and fields, from another class. This feature enhances the reusability of code and facilitates the use of inherited methods while also allowing for the addition of new methods. Inheriting properties creates a parent-child relationship and is commonly referred to as an IS-A relationship.

The concept of inheritance in programming involves various terms such as superclass, subclass, inheritance hierarchy, polymorphism, and encapsulation.

A child class or subclass is a type of class that obtains its features and characteristics from another class, which is known as the parent or superclass. In other words, the child class inherits the properties and behaviors of the parent class.
