.Net Interview Questions

The following is a compilation of frequently asked interview questions and answers regarding .Net, as commonly encountered within various companies. These questions are often asked to assess a candidate's knowledge and proficiency in the subject matter.

1) What is .NET?

.NET is a software development framework that offers a wide range of tools and features for developers. Similar to other development frameworks like J2EE, .NET provides a runtime environment and comprehensive functionalities in the form of class libraries and APIs. It is an ideal platform to create, deploy, and run web services and various other types of applications.

The .NET framework is comprised of three primary components:

  • Common Language Runtime
  • Framework classes
  • ASP.NET

2) How many languages are supported by .NET at present time?

During the initial release of .NET, it provided support for several programming languages such as VB.NET, C#, COBOL, and Perl. As of now, it has expanded its support to almost 44 languages.

3) How is it possible for .NET to support many languages?

When programming in .NET, the code written in a language is compiled to create Microsoft Intermediate Language (MSIL), which is considered as managed code. Since the managed code runs in the .NET framework, the language barrier is eliminated. This means that the code can interact with or use functions from another language after it has been compiled.

4) Is ASP.NET different from ASP? If yes, explain how?

ASP.NET and ASP are two different technologies with distinguishable characteristics. The dissimilarities between these two can be pointed out as follows:

  • Microsoft introduced ASP.NET as a tool for building dynamic web applications, whereas its earlier technology ASP (Active Server Pages) was also a server-side method for designing and creating web pages.
  • ASP.NET and ASP differ in the way they are processed. ASP.NET is compiled, meaning that the code is translated into an executable form by the compiler before it is executed, while ASP is interpreted, meaning that the code is processed line by line at runtime.
  • ASP and ASP.NET both utilize database connectivity to store and retrieve data, but they use different technologies. ASP relies on ADO technology, whereas ASP.NET uses ADO.NET technology.
  • ASP.NET and ASP differ in their approach to object orientation. ASP.NET is fully object oriented, whereas ASP has only partially adopted an object oriented methodology.

5) What is the state management in ASP.NET?

State management is a critical method for handling an object's state across various requests in web applications. ASP.NET offers two types of state management systems. These systems are essential for ensuring smooth functioning and organization of web applications.

  • Client side state management refers to the process of handling and managing the state or data of a web application on the client side, rather than on the server side. It involves storing and manipulating data on the client side to improve the performance and user experience of the web application. This includes techniques such as using cookies, local storage, and session storage to maintain data without having to constantly retrieve it from the server.
  • The process of managing data and information in a server is known as server-side state management. This refers to the storage and retrieval of data by the server, rather than the client-side, and ensures that important data is secured and managed properly.

6) What is the difference between trace and debug?

The Debug class is utilized specifically for debugging purposes during the development process, whereas the Trace class can be utilized for both debugging and release builds.

7) What are differences between system.stringbuilder and system.string?

The major variations between the classes System.StringBuilder and System.String are:

  • System.StringBuilder and System.String are two different classes in C#. While stringBuilder is mutable which means it can be changed or modified, the String class, on the other hand, is immutable and cannot be changed once created.
  • The 'Append' keyword is present in the 'System.StringBuilder' class, but not in the 'System.String' class.

8) What is the difference between int and int32?

Both "int" and "int32" data types are the same in .NET. The term "int" is simply an alias name for the System.Int32 class, which is available in the .NET Framework. Therefore, when you use either "int" or "int32", you are referring to the same 32-bit integer data type.

9) What is the difference between namespace and assembly?

An assembly is a collection of logically related units that are physically grouped together. On the other hand, a namespace is responsible for grouping together classes. Unlike an assembly, a namespace has the ability to encompass multiple assemblies.

10) Explain the differences between value type and reference type.

The key distinctions between value types and reference types are as follows:

  • The fundamental difference between value types and reference types is that the former store their value in the memory location where the variable is defined, while the latter store a reference to the memory location where the value is stored.
  • Memory allocation differs between reference types and value types. Reference types allocate memory in the managed heap, while value types allocate memory on the stack.
  • The content discusses the difference between two types of data structures in programming: reference types and value types. Reference types belong to a class, whereas value types are structured as a combination of data of different types or as an enumeration.

11) What is the difference between session object and application object?

The session object is utilized for managing the session of individual users.

Imagine a scenario where a user logs into an application and receives a unique session ID. If the user logs out or leaves the application, the session ID is removed. When the user returns to the application, a new session ID is assigned to them.

The identity of an object is a unique identifier that distinguishes it from other objects. The ID is usually assigned based on the memory location of the object. However, the ID of an object can change during the lifespan of the program if the object is moved in memory. The ID of an application object, on the other hand, remains constant throughout the duration of the program.

12) What are differences between function and stored procedure in .Net programming language?

The following is an original rendition of the content: Function and stored procedure are both commonly used tools in programming, but their purposes differ. Functions are specialized routines that perform specific tasks and return a single value, while stored procedures are used to perform more complex operations that may involve multiple steps and do not necessarily return a value. In summary, while both functions and stored procedures are useful in coding, they serve different functions and are used under different circumstances.

  • A function can only return a single value, while a procedure has the ability to return one or more values.
  • The select statement can make use of a function, but a procedure cannot be used in this way.
  • A function is characterized by having only input parameters while a procedure may have both input and output parameters.
  • In procedures, exceptions can be managed through a try-catch block, but this approach is not applicable for functions.

14) What is the difference between Hash table and Array list?

A hash table and an array list are two different data structures. The former stores data in a name-value format, whereas the latter stores only values.

To retrieve a value from a Hash table, you have to provide the name as a key. In contrast, to access a value from an array, you have to provide an index number.

An array is used to store data of the same type, while hash tables can store data of different types such as integers, strings, and more.

15) What is the meaning of Immutable?

Immutable is a term that refers to the inability to modify a created thing.

To illustrate, suppose you need to update the value of a variable. In this case, the previous value would be disregarded and a new instance would be created in memory to hold the updated value.

16) What are the advantages of using session?

The following benefits can be acquired by utilizing session:

  • A session is a storage mechanism that allows for the retention of user data and states across an entire application.
  • Storing objects in Python is relatively simple and straightforward. Python provides an efficient, built-in feature known as "pickle" that allows us to store any kind of object with ease.
  • It has the ability to keep individual user data stored in isolated spaces.
  • Sessions provide a secure and transparent user experience since the session object is stored on the server, ensuring the user's data and information remain protected.

17) What are the disadvantages of using session?

The following are the drawbacks associated with utilizing session:

  • When there are many users, the performance of a system can be impacted due to the storage of session data in the server's memory, causing additional processing load.
  • Serializing and De-Serializing session data involves an additional overhead. This is because the object needs to be serialized before storing in the case of StateServer and SQLServer session modes.

Unboxing involves the conversion of a reference type to a value type.

HttpHandler is an interface designed to handle incoming Http requests at a lower level. Its purpose is to service incoming requests, and every incoming Http request received by ASP.NET must be processed by a class instance that implements HttpHandler.

22) What is .NET Framework and what are the main components of it?

The .NET Framework is a platform that enables developers to create and deploy various types of applications, such as console applications, web applications, web services, and window services. It also allows the creation of shareable components for use in distributed computing.

The key elements of the .Net Framework include:

  • Class library
  • The Common Language Runtime (CLR) is a component of the .NET Framework. It is responsible for managing the execution of .NET programs, providing services such as automatic memory management, security, and exception handling.
  • The Dynamic Language Runtime, also known as DLR, is a system that enables dynamic communications between applications and languages in the .NET framework. It allows developers to incorporate dynamic typing and interpreter/COM wrapping capabilities into their applications, making it easier to work with dynamic languages such as Ruby, Python, and JavaScript.
  • Application Domains
  • Runtime Hosts
  • Cross-language interoperability
  • Framework security
  • Profiling etc.

The Manifest is a storage component that holds assembly metadata, which is essential for various functions such as determining which versions of libraries to use, resolving dependencies, and allowing for the security of assemblies.

24) What are the memory-mapped files?

Memory-mapped files enable applications to map the contents of a file to a logical address. This facilitates the sharing of data between multiple processes running on the same machine. To acquire a memory-mapped file object, the MemoryMappedFile.CreateFromFiles() method can be utilized. This method creates a persistent memory-mapped file from a disk file.

26) What is the difference between dispose() and finalize()?

The methods Dispose and Finalize are utilized by CLR for garbage collection of runtime objects in .NET applications. However, there are differences between them.

The runtime automatically invokes the Finalize method, whereas the Dispose method is manually invoked by the programmer.

27) Explain the Code Access Security (CAS) in .NET framework.

The .NET security model is implemented to ensure that unauthorized individuals or processes cannot access resources or perform actions that they are not authorized to do. This is achieved by implementing a security model that includes Code Access Security as one of its components.

28) What is Garbage collection?

Garbage collection is a process that helps prevent memory leaks when executing programs. It involves a background process, known as the garbage collector, that manages the allocation and deallocation of memory for programs. This process is responsible for identifying unreferenced variables and objects in memory that are no longer required by the program. When such objects are identified, the garbage collector frees up the memory associated with them to prevent memory leakage.

37) What is an IL?

The abbreviation IL has multiple expansion forms such as Intermediate Language, Microsoft Intermediate Language (MSIL), and Common Intermediate Language (CIL).

The process of converting .NET source codes involves compiling them to the Intermediate Language (IL) first. The conversion to machine code happens at two possible points - either during installation of the software or at run-time via the Just-In-Time (JIT) compiler.

38) Which method is used to enforce garbage collection in .NET?

The .Net framework provides the System.GC.Collect() method, which can be used to manually trigger garbage collection. This method can be helpful in managing memory and ensuring that unused objects are removed from memory as quickly as possible.

39) What are tuples in .Net?

A tuple is a type of data structure that comprises of a set number of elements that can be of similar or different data types. It is essential to specify the size of a tuple when declaring it, similar to an array.

40) How many elements a tuple can hold?

A tuple is a collection of ordered and immutable elements that can contain from one to eight items. If there are more than eight items, the eighth item can be defined as another tuple. Tuples can be used as parameters or return types in methods.

42) How do you check whether a DataReader is closed or opened?

The "IsClosed" property is used to determine the status of a DataReader. It returns a boolean value indicating whether the DataReader is currently open or closed. If the value returned is "true," the DataReader is closed, otherwise if the value is "false," it is still open.
