1) What is SQL Server?

2) What is Normalization?

Normalization is a process used in relational databases to minimize data redundancy and ensure logical data integrity. This involves dividing the database into multiple tables and establishing relationships between them. By using normalization, performance for the database can be improved significantly.

Types of Normalization

Normalization refers to the process of organizing data in a database to minimize redundancy and dependencies. Various types of normalization techniques are employed to achieve this objective.

  • 1NF
  • 2NF
  • 3NF
  • BCNF
  • 4NF
  • 5NF

There are four types of Normal Forms (NF) in the Relational Database (RD) model, but only the first three are commonly used. The creator of the RD model, E.F Codd, introduced the process of normalization, starting with the first NF and continuing until the third NF.

3) What is De-Normalization?

De-normalization is a strategy used to improve the performance of databases by intentionally adding redundant data. This process involves deliberately introducing redundancy into a table to optimize its performance. De-normalization can enhance read performance but may adversely affect write performance. This approach involves grouping data into redundant forms to achieve better performance. Un-normalized and de-normalized databases are dissimilar, and it's vital to normalize a database before de-normalizing it.

5) What is collation sensitivity? Explain different types.

The concept of collation sensitivity is employed to establish guidelines for organizing and evaluating character data strings. Sorting character data involves adhering to specific rules such as maintaining the correct character sequence, considering character case, accommodating various character widths, and accounting for accent marks, among others.

Various kinds of collation sensitivity:

  • Case sensitivity
  • Accent sensitivity
  • Kana sensitivity
  • Width sensitivity

Case sensitivity is the concept of differentiating characters based on their case. It assigns a unique value to each character, treating uppercase and lowercase letters as distinct entities as they hold different ASCII values in computer programming.

Accent sensitivity refers to the ability to distinguish the differences in pronunciation between accents. It is important to recognize that certain letters and sounds may be pronounced differently depending on the accent being used, such as the letter "a" in American English versus British English. Being sensitive to these differences can aid in effective communication and understanding between people of different accents.

Kana sensitivity is a term used in the Japanese language to describe the distinction between two written scripts: Hiragana and Katakana. It refers to the sensitivity and precision required to differentiate between these two scripts.

Width sensitivity is a feature that allows a system to distinguish between a single-byte character, also called half-width character, and the representation of the same character in double-byte format.

6) What is the standby server?

A Standby server is a type of server that becomes active when the primary server fails and there is a need for continuous availability. In order to ensure server availability, there is a need for a mechanism that can switch the primary server to a secondary or standby server.

There exist three categories of standby servers:

The hot standby method is a redundancy solution where the primary and secondary backup systems operate concurrently to maintain real-time data synchronization. This approach ensures that both systems have identical information.

A warm standby is a redundancy technique where a secondary system operates in parallel with the primary system. The secondary server constantly mirrors data from the primary server, however, there may be some delay in the data synchronization resulting in both servers not having the same data at all times.

A cold standby is a redundancy method where the secondary server is only utilized when the primary server malfunctions. Such systems are suitable for applications that do not involve frequent data changes or for non-critical applications. In cold standby, the standby server physically replaces the primary server.

7) What is the difference between clustered and non-clustered index?

A clustered index is a specific type of index that rearranges the physical storage order of the records in a table. This index provides the sequence in which data is stored in a database. A table can have only one clustered index, and the leaf nodes of this index contain the data pages. Its index ID is 0, and a primary key constraint can automatically create a clustered index.

A non-clustered index is an indexing method where the physical order of the rows on disk does not match the logical order of the index. Unlike clustered indexes, non-clustered indexes store the data and indexes in separate locations. The leaf nodes of a non-clustered index contain index rows, instead of data pages. Furthermore, non-clustered indexes have an index ID greater than zero.

21) What is the DBCC command and why is it used?

Using DBCC, users can monitor and review tables and databases, as well as validate operations performed on the database. A practical example of using DBCC is to ensure that the database is functioning correctly.

DBCC CHECKDB is a command used to verify the linkage between the indexes and tables in a database, ensuring they are accurate and functioning correctly.

DBCC CHECKALLOC is a database command that verifies the correct allocation of all pages in a database. It checks each page thoroughly to ensure that they are allocated appropriately.

DBCC CHECKFILEGROUP is a command that can be used to verify the integrity of all database tables within a specific file group. It scans the file group for any potential errors or inconsistencies, ensuring that the data is consistent and accurate.

Executing the described commands results in the creation of a database snapshot by the database engine while maintaining a consistent transactional state. Checks are then performed against this snapshot, and once the command completes, the snapshot is dropped.

43) Define Magic Tables in SQL server?

A type of table created and managed by SQL server itself to store the updated or inserted values resulting from any Data Manipulation Language (DML) operation is commonly known as Magic tables in SQL server. These tables are extensively used by triggers.

44) What is CDC in SQL Server?

CDC, short for Change Data Capture, is a method used to track any recent modifications applied to a SQL Server table, such as INSERT, DELETE, and UPDATE actions. It records these changes in the table in a format that can be easily recognized and processed by software.
