Top 70+ Most Asked SCCM Interview Questions and Answers

Some common questions asked during technical support interviews are discussed below alongside the best suitable answers.

1) What is SCCM? / What is the full form of SCCM?

SCCM, short for System Center Configuration Manager, is a software management tool created by Microsoft. It enables users to manage Windows-based systems as well as those operating on Linux and Mac OS. Additionally, SCCM is often referred to as ConfigMgr.

2) What are the most important features of SCCM?

SCCM is a widely-used software that has several features worth noting. These features are crucial in managing system configurations, software updates, and security policies.

  • Rewritten: The process of deploying and managing software refers to the activities involved in deploying and monitoring software applications, including installation, configuration, monitoring, and updating.

  • Patch management
  • Desired Configuration Management
  • Asset tracking
  • Reporting
  • Remote control etc.

3) What are SCCM boundaries?

SCCM boundaries are a crucial part of effectively managing devices on an intranet network. These boundaries define specific network locations where devices exist and require management. It's necessary to ensure that each SCCM boundary is unique for each site to ensure precise system center management.

SCCM Boundaries can consist of a variety of elements such as an IP subnet, IPv6 Prefix, IP ranges, Active Directory site name, or an IP address range.

4) What are the different kinds of sites available in SCCM 2007?

SCCM 2007 offers various types of websites for users. These different site configurations serve different purposes and provide distinct functions within the SCCM framework.

  • Child site
  • Parent site
  • Primary site
  • Secondary site

5) What are the various types of sites available in SCCM 2012?

There are three primary categories of websites that are accessible through SCCM:

  • Primary site
  • Secondary site
  • Central Administration site

6) What is the Central Administration Site (CAS)? / What do you understand by a central site?

A Central Administration Site (CAS) or central site is a primary site in the hierarchy of the Configuration Manager that serves as the highest level site. It is responsible for managing all the clients in the hierarchy and executing site management tasks via the console. Only primary sites can serve as child sites for a CAS. The central site database aggregates data from the child sites to the parent site.

7) What is a Site server?

In simple terms, a site server refers to a computer system that has Configuration Manager 2012 or 2007 installed in it.

8) What is the difference between the primary and secondary sites in SCCM?

The Primary site and Secondary site in SCCM have several differences that set them apart. These differences include various factors that distinguish the two from each other.

Primary Site Secondary Site
The main website can offer users the ability to connect to a Microsoft SQL Database. The secondary location is unable to grant entry to the Microsoft SQL Database.
It is possible to allocate customers directly to the main location. It is not possible to directly designate customers to the backup site.
One way to give the Configuration Manager is by using its console. The administration of the treatment is only possible at the main location.

Clients have the authority to directly designate the main sites. Customers do not have the ability to directly designate the subordinate location.

9) What is the child site in SCCM?

A child site is an element of the SCCM structure which receives its data from a parent site. It serves to organize and distribute information in the hierarchy.

11) What is the use of WSUS?

WSUS is a tool that enables system administrators to distribute Microsoft software updates to devices running on the Windows operating system.

12) What do you understand by limiting collection?

To create a new collection, it is necessary to define a limiting collection. This collection determines the subsets of the new collection. Therefore, every collection is a subset of its corresponding limiting collection.

13) What are BITS? Why is it used in SCCM?

BITS stands for Background Intelligence Transfer Service. It facilitates the transfer of data between the SCCM Server and the client, as well as the download of the client to the machine during the client push process.

14) Can a 64-bit operating system support the configuration manager console?

It is possible for a 64-bit operating system to be compatible with the configuration manager console of SCCM, even though the console has a 32-bit architecture.

15) What is the full form of SUP?

The acronym "SUP" stands for Software Update Point. It is an essential part of managing software updates and can be found in the Configuration Manager console as a site system role. Typically, a dedicated site system server with WSUS (Windows Server Update Services) is utilized to create the SUP.

WSUS is a valuable tool for organizations with a large number of Windows devices that require regular updates.

17) What is the difference between the package refresh and the package update?

The package refresh and package update have distinct differences that are important to understand.

Package Refresh Package Update
Refreshing a package typically involves updating the file with the latest version, while updating a package usually involves making changes to the source file.
When refreshing a package, it is possible to replace the existing package files with new ones. In contrast, during an update, the SQL source package version can be updated.
By engaging in a package refresh, whenever a user modifies the file, a compressed file will be generated. On the other hand, in a package update procedure, the creation of a compressed file is not possible.

18) What is the SMS provider in SCCM?

An SMS provider enables us to access and manipulate the Configuration Manager Site database, which can be installed on both the server computer and site database server computer. This functionality allows us to read and write data from the database directly.

During the Configuration Manager setup process, the SMS providers are typically installed on the database server or another server. Following a successful setup, information about the current location of the providers can be found on the general tab in the site properties. These providers are also referred to as WMI providers.
