The purpose of creating a Class Library in .Net is to make our code reusable and adaptable for similar applications. However, it may be limited to specific types of applications. To address the issue of having to write business logic for different platforms, .Net 4.5 introduced the Portable Class Library (PCL). Using this library, developers can write their business logic in a single location and then utilize it across multiple platforms, such as Windows, web, or Mobile, without having to rewrite the code for each platform.

The Portable Class Library (PCL) diagram illustrates the seamless flow between different software applications running on the .Net Framework, each with varying environments and platforms, yet sharing the same business logic through the PCL. 32) How Xamarin.Android applications work?

Xamarin.Android applications rely on the Mono Virtual Machine, developed by Microsoft. Mono is an open-source implementation of the .NET Framework that uses C# and CLR standards. Originally created to enable .Net applications to run on Linux, Mono was modified to support development across a range of devices, including embedded systems. Its first release was in 2001, and it has continued to evolve ever since.

Xamarin utilizes Mono in combination with Android's ART. When working with native applications on Android, certain system functionalities, such as Graphics, Audio, OpenGL, and Telephony, are not directly accessible. However, they can be added through the Android Runtime Java APIs, which can be found in the Java.* and Android.* namespaces. The .NET APIs are used by native applications to interact with these exposed APIs, which in turn utilize the Android Binding to access the Android Runtime Java APIs.

48) What are the triggers? How many types of triggers are available?

In XAML, triggers enable us to specify actions that alter how a control looks based on certain conditions being met for a particular property of that control.

Triggers can be included in the resource dictionary at different levels such as control, page, or application level. There exist four variants of triggers for use:

  • Property Trigger: This trigger executes when the property of the control set a particular value.
  • Data Trigger: This trigger is similar to a property trigger, but it uses the data binding.
  • Event Trigger: This trigger occurs when an event occurs on the control.
  • Multi Trigger: This trigger allows us to set the conditions of multiple triggers before the action occurs.
