Infosys Interview Questions

Infosys is a technology company based in Bengaluru, India that specializes in consulting, information technology and outsourcing services. Under the trade name Infosys Technologies Limited, the company has established a strong reputation as a leading provider of innovative technology solutions.

Infosys has designed its recruitment and selection process to ensure that candidates are selected based on their aptitudes and how well they fit the company's requirements. This approach ensures a successful match between the candidate and the company.

Academic Conditions:

  • Minimum 60% Aggregate in 10th and 12th Mark sheet.
  • A minimum score of 65% or higher in B.Tech. is required.

Company Recruitment and Selection Process

In order to be selected for the Infosys interview, candidates must pass three rounds. The first round is an online test focused on logical reasoning and grammar. The second round is a technical interview where candidates will answer technical questions related to their field of expertise. The final round is the HR interview, which is a common step in many companies' recruitment and selection process.

The first round of Infosys recruitment and selection process evaluates candidates' grammar and IQ through an online test. The test includes questions related to analytical thinking and may vary in content, but the focus remains on language and logical skills. This round is of moderate difficulty and success can be achieved by having good communication skills and IQ level.

For the second stage of the recruitment and selection process, applicants are required to take a Technical Test. A good understanding of technical concepts is necessary for passing this test. As the Technical Test is often the most challenging round, one should pay close attention to the interviewer's questions. It aims to evaluate candidates' technical skills and proficiency, which they have gained from previous work experience.

The third round of the job interview process is typically a face-to-face meeting with a Human Resource Manager. If you have successfully completed the second round, you will be eligible to participate in the HR round. During this round, the HR manager will ask you questions about the company and your personal background. It is important to remain calm and composed during the HR round, as the questions will not be technical in nature. Your communication skills will be tested as you will be required to discuss yourself and the company.

8) Difference between classes and interface?

Below are the distinctions between classes and interfaces:

  • To create an instance of a class, you can create an object of that class. However, interfaces cannot be instantiated because all the methods in an interface are abstract and do not have any implementation. Therefore, there is no need to create an instance of an interface.
  • The declaration of a class involves the use of the keyword "class," while an interface is declared using the keyword "interface."
  • An interface is a type of class that has its own set of members that are used to derive another class. Unlike regular classes that can have public, protected, and private access specifiers for their members, interface members are all declared as public because they are meant to be used by other classes. Therefore, there is no need to specify access modifiers within the members of an interface.
  • Classes and interfaces are both key components of object-oriented programming. While classes declare fields and methods for performing actions on those fields, interfaces only define abstract methods. In other words, classes provide a blueprint for creating objects with specific properties and behaviors, while interfaces establish a set of requirements that classes must adhere to.
  • One key difference between classes and interfaces in object-oriented programming is that a class can inherit from only one superclass, but it can implement multiple interfaces. On the other hand, an interface cannot implement another interface, but it can extend multiple interfaces.
  • A constructor can be defined within a class to initialize the fields of the class, but an interface does not have any fields to be initialized, so it does not have a constructor defined.

Char is a fixed length datatype that stores strings of a specific length, while Varchar is a variable length datatype that can store strings of varying lengths.
