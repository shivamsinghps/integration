DXC Technology Interview Questions

DXC is a global corporation that specializes in providing IT consulting, management consulting, cloud infrastructure, operations, managed services, and outsourcing services to its clients. They are known for their expertise and leadership in the technology industry.

DXC is a globally operating company based in Tysons Corner, Virginia, USA. With a presence in over 70 countries, it is listed on the New York Stock Exchange under the symbol DXC.

The current CEO, President, and Chairman of DXC Technology is John Michael Lawrie.

In 2017, two major companies, CSC and the Enterprise Services division of Hewlett Packard Enterprises, merged to form a new entity called DXC Technology. This new company has since provided guidance and support to some of the world's largest businesses and government entities, helping them navigate through periods of change with success.

3) What is Polymorphism?

Polymorphism refers to the ability to perform a single action in various different ways. The term is composed of two Greek words, "poly" meaning many and "morphs" meaning forms. Therefore, polymorphism literally means many forms.

4) What is the difference between Abstraction and Interface?

Answer:

ABSTRACTION INTERFACE
1. It is possible for an abstract class to contain both abstract and non-abstract methods. In Java programming, an Interface is a type of abstract class that consists of only abstract methods. However, with the introduction of Java 8, Interfaces can now also include default and non-static methods.
2. The feature of multiple inheritance is not supported by abstract classes. The interface has the ability to support multiple inheritance.
3. An abstract class has the ability to provide implementation for an interface. An interface is unable to give the implementation of an abstract class.
4. One can use the abstract keyword to declare a class as abstract. To define an interface in some programming languages, the keyword "interface" is used.

5) What is Encapsulation in Java?

Encapsulation is a fundamental concept in Java programming. It involves packaging both data and code into a single unit, similar to how multiple medications are contained in a single capsule. This practice is essential for protecting data and limiting access to it, allowing for more secure and organized programming.

To ensure complete encapsulation of a class in Java, we can make its data members private. This means that external code cannot access these variables directly. Instead, we can use setter methods to assign values to these variables, and getter methods to retrieve values from them. This way, the class's data is hidden and can only be accessed through its designated methods.

The Java Bean class is an excellent demonstration of a class that is completely encapsulated.

15) What is ER Model?

The ER Model is a conceptual model used in database design. It is based on real-world situations, which include entities and relationships among them. Entities are objects that have a set of attributes that can be represented in a database.
