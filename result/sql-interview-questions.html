SQL Interview Questions and Answers

The subsequent list presents some common and practical interview questions and answers about SQL for both entry-level and seasoned applicants. These queries are constructed to aid you in getting comfortable with the types of inquiries you might encounter during your SQL interview. Typically, interviewers do not intentionally focus on any particular topic during the interview and start with a fundamental comprehension of the topic. Following that, based on your answers, they will continue the discussion.

1) What is SQL?

SQL is a widely used language for managing relational databases that was first created in 1970. It is known for its ability to perform various data manipulation tasks, such as adding, removing and modifying rows. SQL is used for managing data entities and their relationships within a structured database. It is often abbreviated as "sequel."

2) When SQL appeared?

SQL has been around since 1974 and has become one of the most popular languages for managing relational databases. Its usefulness led to it becoming the standard language for the American National Standards Institute (ANSI) in 1986 and the International Organization for Standardization (ISO) in 1987.

3) What are the usages of SQL?

The primary function of SQL is to uphold the relational data and data structures within a database. Here are some typical applications of SQL:

  • One way to interact with a database is by sending queries to it. These queries can be used to retrieve data, modify data, or perform other actions within the database.
  • To obtain information from a database.
  • To add data entries to a database.
  • To modify information contained in a database.
  • One way to remove data from a database is to delete records.
  • To generate fresh databases.
  • To generate fresh tables within a database.
  • One way to generate views within a database is by creating them through appropriate commands and syntax.
  • To carry out intricate tasks on the database.

4) Does SQL support programming language features?

SQL stands for Standard Query Language and is a command language designed for performing database operations such as manipulating, retrieving, updating, and deleting data within the database. Although it is categorized as a language, SQL does not support programming language functionalities such as loop, conditional statements, and logical operations. Its main function is to perform complex operations like joins on the data within the database.

5) What are the subsets of SQL?

There are four important components of SQL that can be classified as subsets.

  • DDL, also known as data definition language, refers to a set of SQL commands used to create, modify, and delete database structures such as tables, indexes, and schemas. It defines the data structure that consists of commands like CREATE, ALTER, DROP, etc.
  • DML, which stands for Data Manipulation Language, is a language used to manipulate data stored in a database. It is used to manipulate existing data in the database. The commands in this category are SELECT, UPDATE, INSERT, etc.
  • Data control language (DCL) refers to a type of computer language that is used to control access to data stored in a database system. Its purpose is to maintain the security and integrity of the data, and it does this by allowing database administrators to set permissions and restrictions on who can access and modify the data. This helps to prevent unauthorized access or tampering with sensitive information. It controls access to the data stored in the database. The commands in this category include GRANT and REVOKE.

11) What is a foreign key?

A foreign key is a type of key used to establish a relationship between two or more tables in a relational database. It is referred to as a referencing key and is designed to link tables by referencing the primary key of another table. This means that a field in one table will refer to the primary key of another table, enabling the identification of unique rows in the referred table and facilitating the maintenance of referential integrity. The relationship between primary keys and foreign keys is critical as it aids with ACID database properties and prevents actions that would jeopardize the relationships between parent and child tables.

13) What is the difference between a primary key and a unique key?

Both primary key and unique key are constraints in SQL that serve important functions. The key difference between them is that while a primary key is used to identify each record in a table, a unique key is utilized to avoid the occurrence of duplicate entries in a column, except for NULL values. This can be further clarified through the following comparison chart:

Primary Key Unique Key
The main purpose of a primary key is to provide a distinct identifier for every individual record within a given table. A unique key serves as an exclusive identifier for data when the primary key is absent from a table.
It is not possible to save NULL values in the primary key field. It is possible to save a NULL value in a column that has a unique key constraint, however, only a single NULL value can be stored in such a column.
It is not possible to alter or remove the values in the primary key column. It is possible to alter the values in the column that serves as the unique key.

22) What is Denormalization in a Database?

Denormalization is a strategy employed by database administrators to maximize the performance of their database system. It involves adding redundant data to a normalized database to simplify complex queries that involve merging data from multiple tables. 23) What are the different types of SQL operators?

Operators are essential keywords or characters used for carrying out specific operations, including those in SQL queries. In SQL commands, operators are utilized mainly within the WHERE clause to filter data based on specific conditions. SQL operators fall into different categories, and their primary purpose is to facilitate data manipulation.

  • Arithmetic operators: These operators are used to perform mathematical operations on numerical data. The categories of this operators are addition (+), subtraction (-), multiplication (*), division (/), remainder/modulus (%), etc.
  • Logical operators: These operators evaluate the expressions and return their results in True or False. This operator includes ALL, AND, ANY, ISNULL, EXISTS, BETWEEN, IN, LIKE, NOT, OR, UNIQUE.
  • Comparison operators: These operators are used to perform comparisons of two values and check whether they are the same or not. It includes equal to (=), not equal to (!= or <>), less than (<), greater than (>), less than or equal to (<=), greater than or equal to (>=), not less than (!<), not greater than (!>), etc.
  • Bitwise operators: It is used to do bit manipulations between two expressions of integer type. It first performs conversion of integers into binary bits and then applied operators such as AND (& symbol), OR (|, ^), NOT (~), etc.
  • Compound operators: These operators perform operations on a variable before setting the variable's result to the operation's result. It includes Add equals (+=), subtract equals (-=), multiply equals (*=), divide equals (/=), modulo equals (%=), etc.
  • String operators: These operators are primarily used to perform concatenation and pattern matching of strings. It includes + (String concatenation), += (String concatenation assignment), % (Wildcard), [] (Character(s) matches), [^] (Character(s) not to match), _ (Wildcard match one character), etc.

29) What is the non-clustered index in SQL?

Non-clustered indexes are indexes that are not the primary indexes (clustered indexes). Unlike clustered indexes, non-clustered indexes are created manually when multiple join conditions and various filters are used in a query. The data in a non-clustered index and the table data are stored separately and therefore cannot physically alter the table's order. Instead, it maintains logical order of the data within the table.

Non-clustered indexes are created for the purpose of searching data within a database. They operate like an index in a book, where the content is in one location and the index is in another. A table can have up to 249 non-clustered indexes. The use of non-clustered indexing can enhance the performance of queries that utilize keys without assigning a primary key.

49) What is the ACID property in a database?

The ACID properties are designed to facilitate transactional processes comprising various tasks. These properties help maintain data consistency before and after a transaction in a database. Moreover, they guarantee that data transactions are handled efficiently and reliably in a database system. In other words, a transaction is a logical sequence of data operations that should comply with ACID properties for optimal results.

The ACID property is a set of characteristics that ensure reliability and consistency in database systems. The acronym stands for Atomicity, Consistency, Isolation, and Durability.

Atomicity is a fundamental concept in database management. It guarantees that all statements or operations within a transaction will be executed successfully. If any part of the transaction fails, the entire transaction will fail, and the database state will remain unchanged. This concept is implemented using several features such as COMMIT, ROLLBACK, and AUTO-COMMIT.

Consistency is a crucial property of a database system. It ensures that all validation rules are met before allowing changes to be made to the database. Simply put, the database remains in its original state unless the transaction is committed successfully. Consistency protects data from potential crashes and ensures that all data within the system remains valid and accurate.

The isolation property in database management systems refers to the ability of the transactional unit to operate independently and transparently from other executing statements. Its primary purpose is to regulate concurrency and prevent data corruption in a database. Essentially, it ensures that concurrent transactions do not interfere with each other, and their execution remains isolated from one another.

Durability is a crucial feature of a database system that ensures the committed transactions remain intact and permanent even in scenarios like power outages, system crashes, or other failures. This property guarantees the data's reliability, and users can retrieve it without fear of loss or corruption.
