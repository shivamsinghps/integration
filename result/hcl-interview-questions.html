1) Does every class have a Constructor?

It is necessary for every class to have a constructor, either in the form of a parameterized or default constructor. In the absence of a user-defined constructor, the default constructor is automatically added to the code.

Constructor is used to perform object initialization in programming. It is specifically designed to initialize an object. The main purpose of developing constructors is to provide a way to initialize the different attributes and properties of an object.

2) How Java enable High Performance?

Java achieves high performance by utilizing a Just-In-Time (JIT) Compiler. This program converts Java bytecode, which is a program containing instructions that need interpretation, into instructions that can directly be sent to the processor.

3) Why is java considered as dynamic?

Java has been designed in a way that allows it to adapt to changing environments. It has the ability to carry a large amount of runtime data that can be used to confirm and manage object access during runtime.

4) What is joins in SQL?

Joining is the process of linking multiple tables to retrieve data from different databases.

When a developer wants to prevent other classes from instantiating an object, he/she uses a Private Constructor. By restricting object creation to a single class, developers gain greater control over object interactions and the functioning of the code.

Primary Key Unique Key

7) Differentiate between Method Overloading and Method Overriding.

Method Overloading Method Overriding
One benefit of using Method Overloading is that it increases the clarity and organization of a program's code, making it easier to read and understand. Method Overriding is a programming technique that involves providing a new or modified implementation of a method that already exists in the superclass. This is used to tailor the behavior of the method to the specific needs of the subclass. When using Method Overloading, it's necessary to have different parameters for each method. Method Overriding requires that the parameters in the overriding method are identical to those in the original method. The process of overloading occurs during the compilation phase of software development. The process of overriding occurs during the program's runtime. When overloading a method, the return type is not a critical factor and can be the same or different from the original method. Method Overriding requires that the return type should be identical. Possible rephrased version: Overloading in Java allows developers to define multiple methods with the same name but different parameters. This feature is not limited to instance methods and can also be applied to static methods, final methods, and even private methods. However, it's worth noting that private methods can only be called within the same class, and final methods cannot be overridden by subclasses. Additionally, while static methods don't belong to any instance and can be called without creating an object, they still belong to a class and can be overloaded based on the number, order, and type of their parameters. In Java, it is not possible to override a method that is static, final, and declared as private in the superclass.

8) What is the purpose of finalize () method in Java?

The finalize() method in Java is an important method that is similar to the main method in Java. This method is executed just before the garbage collector reclaims the object, providing a final opportunity for the object to perform any necessary cleaning activities, such as releasing any held resources or terminating open connections.

9) Why can static method not override?

When it comes to overriding methods in Java, it's not possible for a static method to override an instance method. This is because static methods are associated with the class, whereas instance methods are associated with an object. Therefore, static methods exist in the class area while instance methods exist in the heap area.
