Most Asked Active Directory Interview Questions

Here are some common questions that are often asked in an Active Directory interview, along with their suggested answers.

1) What do you understand by the term Active Directory?

Active directory is a metadata or index structure utilized in Microsoft Windows servers and computers to store information and data about networks and domains.

Active Directory serves as a central database that stores a range of important network information, including user details, computer information, and other network object data. It is a powerful tool that enables comprehensive network management and administration.

2) What is a domain?

A domain is a collection of network resources that are accessible to a group of users who can easily access them by logging in to the domain. These resources may be located on different servers within the network. The domain is identified by a unique system address, which provides valuable information. The idea of a domain was first introduced in Windows NT, and it allows users to access various computer resources using a single combination of username and password.

3) What is the default protocol used in directory services?

LDAP is the default protocol utilized in directory services for non-payment default purposes. This acronym stands for Lightweight Directory Access Protocol.

4) What is the difference between domain local, global and universal groups?

Domain local groups are groups with a scope that is limited to a specific domain, and they are used to assign permissions to resources within that domain. The key difference between domain local groups and global groups is that domain local groups only allow members from the same domain to be added and permissions can only be assigned within that domain. This makes them ideal for assigning permissions to assets that are specific to a particular domain.

5) What is the Sysvol folder? Why is it used?

The Sysvol folder plays a critical role in storing a server's version of a domain's public files and distributing policy and logon scripts to all domain members. It is also responsible for replicating group policies from one domain controller to others within a specific domain.

6) Can a user create a new universal user group?

To create a new universal user group, all domain controllers must be upgraded to Windows Server 2003 Active Directory in a native-mode environment. This type of group is only compatible with Windows Server 2003 and cannot be used in earlier versions.

7) What do you understand by the term "Forest" in AD?

A forest refers to a group of Active Directory (AD) domains that have their own distinct schema within the AD. This schema is shared by all domain controllers (DCs) within the forest and is replicated in a hierarchical manner among them.

8) What is the difference between enterprise admin group and domain admin group in AD?

The following explains the distinct features of enterprise admin group and domain admin group in AD:

Enterprise Admin Group Domain Admin Group
The Enterprise Admin Group holds full authority over every domain present in the forest. The Domain Admin Group holds full authority over the domain, granting its members complete control.
By default, the group of administrators for an enterprise is included in the Administrators group on all of the domain controllers within the forest. Typically, when a domain is connected, the Domain Admin Group is automatically added to the Administrators group on all member servers, workstations, and domain controllers.
It is important to exercise care when adding new users to this particular group as they have total authority over the forest. It is important to exercise caution when adding users to this group as it has complete control over the domain.

9) What is ARP? What do you understand by ARP Cache Poisoning?

ARP is an abbreviation for Address Resolution Protocol. It's a communication protocol used to link an IP address to the corresponding physical machine address within a network.

ARP Cache Poisoning is a method through which a table known as the ARP cache is utilized to maintain a connection between a MAC address and its linked IP address. The ARP protocol outlines the guidelines for making this connection and facilitating address conversion in both directions.

10) What do you understand by Kerberos?

Kerberos is a network authentication protocol designed to provide secure authentication for client applications through the use of secret-key cryptography. Its main purpose is to ensure secure verification for clients and maintain the integrity of the network.

11) What is LSDOU?

The LSDOU model is a method of inheriting group policies, which involves applying policies to various levels, including local machines, sites, domains, and organizational units. The name LSDOU represents each level where policies can be applied.

12) What is Active Directory Schema?

The Active Directory Schema serves as the framework for an Active Directory Network. It outlines the structure and specifications for all objects within the network. Every object created in an Active Directory Network is associated with the Active Directory Schema, which defines its type.

13) What are the different components of Active Directory Schema?

An AD Schema is composed of three key elements:

  • Objects: Objects are the entities that Active Directory uses to store information about the resources in its network. In Active Directory, objects are used as references to the resources in the network. For example, if a computer belongs to the Active Directory network, there would be a computer object that will represent this computer in real life. The computer object will also store some information about the computer such as operating system, running status, etc.
  • Classes: In an Active Directory Schema, attributes are grouped and categorized into object classes. Object classes are, therefore, hierarchical classifications of object definitions. There are three types of classes in an Active Directory schema:
    1. Abstract class
    2. Structural class
    3. Auxiliary class
  • Attributes: Attributes are the entities that are used to store information about the objects in the Active Directory environment. For example, a user object can have attributes such as their first name, last name, telephone number, address, and other attributes that store data about the user. Some attributes also store information about other attributes than the object itself. There are several types of attributes, such as linked attributes, indexed attributes, and global attributes.

14) What are the newly added features in Active Directory (AD) of Windows Server 2012?

The following are a collection of enhancements that have been introduced in the Active Directory of Windows Server 2012:

  • Rewritten: One of the new features in Windows Server is the updated dcpromo tool, used for promoting a server to a domain controller. The tool includes an improved wizard interface for easier setup and management of domain controllers. It facilitates users to view all the steps and review the detailed results during the installation process.
  • Enhanced Administrative Center: In the newer version of Windows 2012, the administrative center is well designed compared to the earlier version of the active directory. The exchange management console is well designed.
  • The graphical user interface (GUI) has been applied to the recycle bin. The Windows Server 12 provides so many ways to enable the active directory to recycle bin through the GUI in the Active Directory Administrative Center, which was not possible in the earlier version.
  • Fine-grained password policies refer to a set of rules and restrictions set by administrators to dictate how users should create and manage their passwords. These policies usually go beyond standard password requirements, such as length and complexity, and may include factors such as password age, history, and uniqueness. FGPP is an essential security feature that helps organizations safeguard against cyber threats and improve their overall security posture. Implementing FGPP is much easier in Windows Server 12 than in the earlier versions. It also facilitates us to create different password policies in the same domain.
  • A tool for viewing the history of Windows Power Shell commands is available. It allows users to view past commands and their results within the Power Shell command line interface. It also provides Windows PowerShell commands related to the actions executed in the Active Directory Administrative Center UI.

15) What system state data contains?

The information stored in the system state data comprises of the following elements:

  • Registry
  • Startup files
  • System files
  • Registration Database
  • Memory page file
  • AD information
  • SYSVOL Folder
  • Information about cluster service, including its features and capabilities.

16) What are the main components of Active Directory?

Active Directory is comprised of two primary elements:

  • Physical Structures: It contains the Domain controller and Sites.
  • Logical Structure: It contains the Trees, Forest, Domains, and OU.

17) What do you understand by Tombstone lifetime?

The Tombstone lifetime feature is crucial in keeping track of how long a deleted object will remain in Active Directory. When an object is deleted in Active Directory, it is stored in a designated object referred to as TOMBSTONE. By default, the Tombstone lifetime in Windows lasts for 60 days, unless configured differently in the forest configuration settings.

18) What do you understand by a child DC or CDC?

A child DC or CDC refers to a subdomain controller that operates under the main or root domain controller. This controller is utilized to allow for the sharing of a namespace.

19) What is the full form of APIPA? Why is it used?

APIPA (Automatic Private IP Addressing) is a feature that is commonly found in Windows operating systems such as Windows 98, 98 SE, Me, and 2000. Its purpose is to automate the allocation of an Internet Protocol address to a computer that has this feature installed.

20) What do you understand by RID Master?

The RID master plays a crucial role in assigning distinct IDs to all objects created in the Active Directory. These IDs are referred to as Relative Identifiers.

21) What do you understand by Infrastructure Master?

The Infrastructure Master is responsible for updating references from objects within its domain to objects in other domains. A domain can have only one Infrastructure Master DC. This means that it plays a crucial role in managing information related to user and group accounts, as well as the global catalog.

22) What do you understand by organizational units?

An Organizational Unit is an essential aspect that has implications for policy, security, skills, and management. It is a type of container in LDAP that can be represented as a sub-domain component with similar characteristics to domains.

23) What is the use of Active Directory Recycle Bin?

The Active Directory Recycle Bin is an attribute exclusive to the Windows Server 2008 AD system. Its primary function is to restore accidentally deleted Active Directory objects without relying on backed-up AD databases or restarting domain controllers.

24) What do you understand by domain trees and forests?

Active Directory involves two critical concepts - domain trees and forests. Essentially, a domain tree refers to a group of one or more domains that share a common namespace. For instance, domains such as jobs.career.com and career.com are considered part of the same domain tree. Note that if a created domain does not contain the full name of the parent or forest root domain, it's automatically distinguished as part of an independent domain tree.

A forest consists of multiple domain trees that share a common namespace. It is possible for the domains in two separate domain trees, such as career.com and colleges.com, to be part of the same forest. The name of a forest is derived from the first domain created within it. For instance, if the first domain to be created is career.com, then the forest will be referred to as career.com. Additional domains, such as jobs.career.com and colleges.com, can be created within the forest at a later stage.

25) What is the purpose of replication in Active Directory?

Replication is essential for disseminating data stored in the index across an organization, thereby enhancing data availability, performance, and security.

26) What are the different ports used by Active Directory?

The following is a collection of ports that are used by Active Directory for communication purposes.

  • RPC endpoint mapper: port 135 TCP, UDP
  • NetBIOS name service: port 137 TCP, UDP
  • NetBIOS datagram service: port 138 UDP
  • NetBIOS session service: port 139 TCP
  • SMB over IP, also known as Microsoft-DS, is a protocol used for file and printer sharing between computers on a network. port 445 TCP, UDP
  • LDAP: port 389 TCP, UDP
  • LDAP over SSL: port 636 TCP
  • Global catalog LDAP: port 3268 TCP
  • The topic of discussion is Global Catalog LDAP over SSL. This involves the use of secure communication protocols to establish a secure connection between an LDAP client and the global catalog server. The implementation of SSL ensures that the communication is encrypted and secure, thus protecting sensitive information from unauthorized access. port 3269 TCP
  • Kerberos: port 88 TCP, UDP
  • DNS: port 53 TCP, UDP
  • WINS resolution: port 1512 TCP, UDP
  • WINS replication: 42 TCP, UDP
  • RPC: Dynamically-assigned ports TCP, unless restricted

27) What are the best tools that you would like to use to edit AD?

The Adsiedit.msc is a powerful tool designed to help Active Directory administrators execute low-level editing tasks. It features a graphical user interface to assist administrators in adding, editing, and removing objects in the directory service. Adsiedit.msc is a Microsoft Management Console snap-in that uses Application Programming Interfaces to interact with the Active Directory.

28) Which program is used to manage trust relationships from the command prompt?

Netdom.exe is a command-line utility tool designed to manage and manipulate Active Directory within Windows environments. It enables system administrators to handle trust relationships within Active Directory through a command prompt. Furthermore, this essential application simplifies the process of joining computers to domains for managing trusts in batches. It is also used to verify trusts and secure Active Directory channels for optimum functioning of the system.

29) What is the full form of SID? Why is it used?

A SID, which is an abbreviation for Security Identifier, is a distinctive identifier of variable length that is used to identify a trustee or security principal.

30) What do you understand by PDC emulator? How would one know whether the PDC emulator is working or not?

PDC Emulators serve an essential function within a domain by managing the synchronization of time across the network. It acts as a decisive factor in case of any conflicts and acts as a failover mechanism in case of authentication failure. There is only one PDC emulator in every domain that takes care of these critical functions.

The following factors can help determine the functionality of the PDC emulator:

  • If the clock on your device is not displaying the correct time in sync with the correct time zone, you may need to adjust the settings.
  • If the accounts of the user are not blocked or suspended.
  • If the Windows NT Backup Domain Controllers (BDCs) are not receiving updates, it means that they are not being kept in sync with the Primary Domain Controller (PDC) and may potentially cause discrepancies in the network.
  • If computers that are running on systems before Windows 2000 are incapable of altering their passwords.
