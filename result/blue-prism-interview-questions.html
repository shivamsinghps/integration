1) What is Blue Prism?

Blue Prism is a company that specializes in automation software development. They offer a popular RPA tool that helps organizations streamline their processes. The term "Robotic Process Automation" was coined by Blue Prism and is now widely used across the industry. Blue Prism is a UK-based company and is seen as a leader in the field of RPA.

Blue Prism is a software tool that has been developed to create bots or software robots. These bots are programmed to perform predefined tasks which are usually repetitive and rule-based, enabling businesses to automate routine tasks and reduce human intervention.

2) What does RPA stand for?

RPA, or Robotic Process Automation, is a type of automation that employs software robots to simulate human tasks. These tasks commonly involve manual, repetitive, and rule-based processes. With RPA, businesses can automate their digital processes across multiple systems and applications without needing to modify their existing IT infrastructure or systems.

Robotic refers to a collection of programming instructions that emulate human movements and action, similar to that of a robot.

A process refers to a series of actions or steps that are carried out in a specific order to achieve a particular goal or outcome.

Automation refers to the use of software robots to perform tasks independently, without the need for human involvement.

15) What are the main phases of Blue Prism Life-cycle?

The Blue Prism Life-cycle is comprised of three primary stages.

During the first phase of automation development, developers collaborate with testers to ensure that the automation solution meets the requirements outlined in the Process Definition Document (PDD). This is an important step in creating a successful automation solution.

Phase 2 involves the use of Process Studio to develop process diagrams in accordance with the specifications outlined in the PDD. At this stage, tests are also conducted to evaluate the system's efficiency and functionality in a live data testing session.

Phase three encompasses the following:

  • The last automation solution undergoes testing to ensure its efficiency. Once it passes the testing phase, it is implemented within a controlled environment for further execution.
  • The management of the overall performance of the process is overseen, and any additional features are incorporated into it.
  • The functionality is tried and evaluated in a genuine setting within the Blue Prism Control Room.
  • The test report and metrics are produced and compared against the agreed-upon acceptance criteria outlined in the PDD. Once the process has been implemented to satisfy these requirements, it can be launched into production.

21) Define these terms: IPA, PDD, FRQ, SDD, OID, PDI, and ODI.

IPA stands for Initial Process Analysis, which is a comprehensive evaluation of the process solution. This analysis covers the necessary conditions, time and resources required for development, and suggested solutions for the production environment needs.

A PDD, or Process Definition Document, is a document that outlines the manual procedures that will be digitalized. It includes a flowchart of the process and thorough explanations of each stage according to the company's needs.

FRQ stands for Functional Requirements Questionnaire and it is a brief document designed to collect important information and identify areas that require attention. It includes checklists and other relevant areas that need to be considered.

SDD is an abbreviation for Solution Design Document and it builds upon the inputs provided by PDD and FRQ. The primary objective of SDD is to automate the processes detailed in PDD, resulting in an end-to-end solution by Blue Prism.

An "Operational Impact Document" or OID is a crucial first step once an automation solution has been delivered to a client. This document outlines the responsibilities of the client's operations team in operating and maintaining the solution. The OID serves as a guide for the operations team to ensure the smooth operation and optimal performance of the automation solution.

PDI, also known as Process Design Instruction, is a vital element in the development of automation processes. It serves as a blueprint for designing or creating the processes and contains detailed information that is not included in SSD. PDI offers increased clarity throughout the project and helps in achieving the desired outcome.

ODI is an acronym for 'Object Design Instruction'. This is a detailed plan that aids in the creation of business objects.

22) Explain work queues in the context of RPA Blue Prism. How can we configure priorities for work queues?

Blue Prism uses Work Queues to efficiently process large volumes of work by breaking them into smaller tasks. This method is highly effective in deploying bots and speeding up the process. Multiple bots are utilized to minimize the time taken for the separation of work into smaller tasks, making it the preferred approach for such operations.

When working in queues, it is necessary to determine the order in which tasks are completed. This involves setting priorities for each item on the work queue. To configure the priority, specific steps must be followed.

  • To begin, we must utilize the "Add to Queue" function to append an item to the appropriate queue.
  • After creating a new process in the queue, the next step is to modify the start-priority of the item. This will involve working with the existing queue item to set the priority level for the new process that has been created.
  • After adding an item to the queue, we can assign a priority value to it using a straightforward process. This can then be repeated with additional items, however, it's important to note that once a priority is given to an item, it cannot be modified later.

32) Why do we use the Wait stage in Blue Prism?

The Wait stage is a crucial part of the process as it allows for the absorption of potential system delays and ensures that the process follows the correct path. Typically, waiting at the beginning of an action confirms the accuracy of the process, increasing its resilience. By default, the Wait stage has a 5-second timeout, which means that the process will wait before taking any action until the specified timeout has elapsed.

When a timeout occurs, it is recommended to throw an exception instead of attempting to retrieve the process through the Wait stage. It is important to let the process handle the exception itself, as it may have procedures in place such as system restarts, alerts, or timed retries to handle the situation. This approach is more effective than trying to manually handle the timeout.

It is important to wait for the screen to change when navigating to any stage that may cause an update. By implementing wait stages, you can increase speed without incurring any latency. This is a key strategy for improving overall performance.

33) What is meant by Throttle?

A throttle is a technique used in wait stages within a workflow, which involves inserting a pause between stages. This pause is not triggered by a specific wait condition. Instead, it typically uses a data item to determine the length of the timeout.

35) What is the main difference between Match Index and Match Reverse options in Blue Prism?

The Match Index and Match Reverse options are two techniques used to search for an element. The former is applied when the search is done from top to bottom, while the latter is used when searching from the bottom to top. When using these options and the desired element is found, the search stops immediately, and all further duplicates are ignored.

In situations where it is necessary to locate a particular element, using the "id" or "class" attributes can be quite advantageous. These attributes help to minimize the time spent searching for elements. However, it is important to note that they should only be utilized when there is no need to identify duplicate elements or when the probability of duplicate elements being present is unlikely.

39) Define System Unavailable Exception and Mark Exception.

System Unavailable Exceptions occur when an application fails to launch, most commonly due to unresponsive or crashed software. These situations arise when the application's design and development did not account for such unexpected crashes.

Whenever a new item is generated, its unique ID is assigned to the output parameter through the use of the Mark Exception. To ensure full control over the newly created item, the Keep Locked indicator is usually enabled.
