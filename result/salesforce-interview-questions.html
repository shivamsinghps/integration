8) What is Master-detail relationship?

The Master-detail relationship is a close connection between different objects in Salesforce, where the parent record governs the behavior of the child record with regards to visibility and sharing. If the master record is deleted, so are all the child records associated with it. Additionally, the security settings of the parent object extend to the child object.

Example:

If a Master-detail relationship is established between the objects "party" and "people", with "party" as the parent object and "people" as the child object, then deleting a "party" record would result in the deletion of all associated records as well.

A master-detail relationship between two objects in Salesforce allows the creation of a roll-up summary field on the master object. This field is unique to this relationship and is used to summarize data from the detail records.

A Roll-up summary is a feature in Salesforce which enables us to compute values pertaining to child records, such as count of child records, average, total, etc., that are associated with a parent record.

9) What is the lookup relationship?

A Lookup relationship in Salesforce creates an association between two objects, where one object is the parent and the other is the child. The relationship is not strongly coupled, as both parent and child objects have their own sharing settings and security controls. This means that if a parent record is deleted, the child records will not be automatically deleted with it.

11) What are some Governor limits in Salesforce?

Governor limits regulate the storage of data and records in shared databases due to the multi-tenant architecture of Salesforce. This means that multiple customers' data is stored in a single database. As a result, Salesforce puts a cap on the amount of data that each customer can store.

Salesforce has implemented Governor limits to ensure equal sharing of resources among all users and prevent monopolization. These limits were introduced to maintain fair usage of shared resources.

One of the major hurdles faced by Salesforce developers is governor limits. These limits restrict the amount of resources that can be utilized by the apex code. If these limits are exceeded, it can result in a runtime exception that cannot be rectified. Therefore, it is imperative for developers to exercise great care while developing applications for Salesforce.

The following is a compilation of notable Governor restrictions.

  • Pre-transaction Apex limits
  • Static Apex limits
  • Size-Specific Apex limits
  • Miscellaneous Apex limits

38) How does Apex work?

The programs of Apex can only be executed on demand and specifically on the force.com platform.

Initially, the apex code is transformed by the application server into a group of abstract instructions which can be interpreted by the apex runtime interpreter.

Once the process of compiling code is complete, the resulting code is saved to metadata.

When the end users trigger the execution of Apex by clicking on a button or Visualforce page, the compiled instructions are fetched from the metadata by the application servers. Subsequently, these instructions are sent to the runtime interpreter, which processes them and returns the output to the users.
