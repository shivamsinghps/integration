Ansible Interview Questions

Here are some commonly asked questions and answers for an interview on Ansible.

1) What is Ansible?

Ansible is a software tool used for automating various IT operations such as cloud provisioning, application deployment, intra-service orchestration, and more. It is a freely available open-source platform that offers IT professionals an effective way to manage their IT infrastructure.

Deploying Ansible is a simple process due to its agentless architecture. It doesn't require any custom security infrastructure on the client-side and operates by sending modules to the clients. These modules are then executed locally on the client-side, and the data is sent back to the Ansible server. Connecting to clients is also straightforward with the use of SSH keys.

2) What are the uses of Ansible?

  • Ansible is available to everyone without any cost.
  • Ansible stands out for its lightness and consistency, allowing for versatile usage regardless of operating systems or hardware.
  • It is very secure due to its agentless capabilities and open SSH security features.
  • You don't require specialized skills in system administration to install and use Ansible.
  • Ansible is easy to learn thanks to its well-documented structure and configuration. As a result, users can quickly pick up the necessary skills and easily maneuver through the platform.
  • Its modularity regarding plugins, inventories, modules, and playbooks make Ansible perfect companion orchestrate large environments.

3) What is Ansible Tower?

Ansible Tower is an elevated version of Ansible, designed for enterprise-level use. It is a web-based platform that offers a user-friendly interface to manage organizational tasks, including a comprehensive dashboard that displays state summaries of all hosts, ensuring efficient deployment and configuration monitoring.

4) What is Ansible Galaxy?

Ansible Galaxy is a platform where role-sharing is possible. It offers a command-line tool that lets users manage, install, and create roles.

Ansible Galaxy is a platform that enables users to share and explore reusable roles and application installation scripts for server configuration using Ansible. It's a great way to discover exciting features and collaborate with other users in the community. Many people actively contribute and share their roles on Ansible Galaxy.

5) What is Ansible Modules?

Ansible modules are independent pieces of programming that can be utilized in individual command line operations or in tasks within a playbook. They provide specific functionalities and commands that can be used to automate different processes.

In Ansible, there are certain components known as modules which can also be referred to as library plugins or task plugins.

Ansible comes with a set of modules known as the module library. These modules can be executed remotely on hosts through a playbook or even directly.

It is possible for users to create their own modules that can perform various tasks such as managing system resources, files, packages, and services. These modules can also handle the execution of system commands.

6) How does Ansible work?

Ansible operates by establishing connections to nodes and deploying a lightweight program known as Ansible modules to them. These modules are then executed by Ansible and eliminated upon completion. The modules library can be located on any system, and there is no need for any servers, databases, or daemons.

The central controlling node in Ansible is known as the Management Node. Its role is to oversee the entire playbook execution process. To determine the target hosts, Ansible uses inventory files. The Management Node creates an SSH connection to each host and runs small modules to install or configure software on that machine.

  • In order to use Linux on a Windows system, it is necessary to set up a virtual machine that has Linux installed on it.
  • Python-Conda is a package and environment management system, which helps in managing multiple software packages and their dependencies. It facilitates easy switching between different versions of packages and helps in creating isolated environments for different projects. It is mandatory to have Python version 2.6 or higher to use Conda.

8) What are the variables in Ansible?

Variables in Ansible are a powerful feature that allow you to assign values to a variable and use it anywhere in your playbook. You can also add conditional statements around the variable values, allowing for more flexibility in how you use them throughout your playbook. Essentially, variables work much like programming language variables, and are a crucial tool for efficient and effective playbook execution.

9) What is Ansible Task?

An Ansible Task serves as a way to divide configuration policy into smaller chunks, which can be automated using code blocks.

10) Explain the basic terminologies or concepts in Ansible?

Ansible is a popular tool in the world of IT automation. There are several key terms that are commonly used when working with Ansible. These terms are important to understand in order to effectively utilize Ansible's many capabilities.

  • Controller Machine: The Controller machine is used to provisioning the servers, which is managed. This is the machine where Ansible is installed.
  • Inventory: An inventory is an initialization file which has details about the different servers you are managing.
  • Playbook: It is a code file that is written in the YAML format. A playbook contains the tasks that need to be automated or executed.
  • Task: Every task represents a single procedure that needs to be executed, e.g., installs a library.
  • Module: A module is the set of tasks that can be executed. Ansible has 100s of built-in modules, and also you can create custom ones.
  • Role: The role is a pre-defined way for organizing playbooks and other files to facilitate sharing and reusing portions of provisioning.
  • Play: The task executed from start to finish, or the execution of a playbook is called the play.
  • Facts: Facts are global variables which are store details about the system, such as network interfaces or operating system.
  • Handlers: Handlers are used to trigger the status of a service, such as restarting or stopping a service.

One can conceal sensitive information in verbose output while maintaining its completeness by using a specific command.

One way to use the no_log attribute in Ansible is by including it in a task to prevent sensitive data such as passwords and keys from being logged. It can also be used at the playbook level to apply the no_log attribute to the entire play.

14) What are Ansible Vaults, and why are they used?

Ansible Vault is a security feature that enables secure storage of confidential information. It offers the ability to encrypt various types of data such as complete files, YAML scripts and variable sets. The feature provides an option to integrate sensitive data into your playbooks while ensuring that it is protected from unauthorized access.

Ansible Vault offers file-level granularity, which means files can either be encrypted or unencrypted entirely. The encryption and decryption of data in Ansible Vault utilizes a single password approach, making it easy for users to manage and secure their data.
