Linux operating system internals

1) What is Linux? Discuss its features.

Linus Torvalds created Linux, an operating system that is based on the UNIX system. It is an open source operating system that was developed to provide users with a low-cost and free alternative.

Linux is a computer operating system that is similar to Unix and manages a system's hardware resources, such as memory, storage, and CPUs, while also facilitating communication between software and hardware. It was first released on October 5, 1991, and is known for being faster and more secure than Windows. It is an open-source software that is based on the Linux Kernel and can be downloaded and installed on various devices such as computers, laptops, and mobile phones. Linux offers a variety of operating system options, including Gentoo, SUSE Linux, Debian, and Ubuntu.

Key elements of the Linux operating system include:

  • Adaptable and robust: Linux can work for prolonged times without crushing and is very much resistant to security threats.
  • Open-source and free: it is easily and freely available to everyone.
  • More secure: Linux is more secure because it offers security with authentication features, such as file system access control, security auditing, and password authentication.
  • Multiprogramming system: Multiple functions or an application can simultaneously run.
  • Application support: Linux includes its software repository, which is a storage location or place from where software can be redeemed so that users can get applications.
  • A graphical user interface (GUI) is a type of user interface that utilizes graphical elements such as images, icons, and menus to facilitate interaction with electronic devices.Linux offers an interface for users to collaborate with the system and permits to use of graphical user interface applications, like Firefox, VLC, etc.
  • Customized keywords: There are several different languages used across the world. Hence, Linux supports multiple language keyboard installation.

2) What is the difference between UNIX and Linux?

Initially, UNIX was developed as a proprietary operating system exclusively for Bell Laboratories. Subsequently, a commercial version was released. On the other hand, Linux is an open-source, non-proprietary operating system made available to the general public for free.

3) What is Linux Kernel? Discuss its functions.

The Linux Kernel is a type of software that operates at a low-level, serving as a resource manager for hardware functionality. It acts as an interface for users to interact with the system at a higher level.

The Linux kernel is a vital part of the Linux operating system because it serves as a resource manager that connects hardware and software. Its primary function is to manage hardware resources and provide an interface for users to interact with. The kernel is the first program loaded when a system boots up and is considered to be a low-level system software.

The main purposes of the tool are:

  1. Storage management
  2. Device management
  3. Process management
  4. Memory management
  5. The ability to control access and usage of different computer accessories.

5) What is LILO?

LILO is a program that is used to load the Linux operating system onto a computer's main memory, allowing the system to begin its operations. It is a tool that plays the role of a bootloader, enabling a dual-boot system. LILO serves as the intermediary between the computer's hardware and the Linux operating system. It performs several functions, such as starting the kernel, locating and loading memory, and recognizing other software components. To run the Linux operating system, it is necessary to install LILO as the bootloader since it is responsible for initiating the boot process.

6) What is the advantage of open source?

Open source software provides the unique opportunity for individuals to freely distribute their software, with open access to their source code. This allows for users to not only add new features but also debug and correct any errors in the code.

7) What are the basic components of Linux?

Linux, like other operating systems, is made up of several parts including the kernel, shells, GUIs, system utilities, and application programs. These parts work together to ensure the smooth running of the operating system. There are five primary components of Linux, which are described below.

  1. Kernel: It is the main or core Linux part and is generally liable for every major activity of the operating system, such as device management, process management, etc.
  2. System library: System libraries are unique programs or functions with which system utilities or application programs can access kernel features without code requirements. It is used for implementing the functionality of the operating system.
  3. System utility: System utilities are utility programs that are liable to perform individual-level and specialized tasks. These are considered more responsible and permit users to handle the system.
  4. Hardware: It's physical hardware that contains items like a CPU, display, keyboard, mouse, etc.
  5. Shell: It's an environment where we can execute our commands, programs, and shell scripts. It's an interface between the kernel and the user that hides every complexity of the kernel functions from the user.

41) What are the Process States?

The states of a Linux process are a series of distinct stages that a process can go through from its beginning to its end. Linux processes are capable of existing in several different states. These states represent the current condition or activity of the process as it executes. Examples of Linux process states include:

  1. New/Ready: A new process is made and is ready to execute.
  2. Running: The process is being run.
  3. Blocked/Wait: The process waits for input through the user, and if it does not have resources to execute, such as input, file locks, and memory, it can remain in a blocked or waiting state.
  4. Completed/Terminated: The process has been terminated by the operating system or finished the execution.
  5. Zombie: The process is aborted, but information related to the process is still available and is available within the process table.

44) What do you mean by LVM, and why it's important?

LVM is a useful tool that provides benefits such as improved abstraction, control, and adaptability. It also allows for flexible disk space management and online resizing of file systems. 48) What do you mean by a Zombie Process?

A zombie process, also called a dead or defunct process, is a type of process in Linux. It occurs when a process finishes its execution, but its data remains in the process table. This can be caused by a lack of communication between the child and parent processes. The child process may become a zombie process because the parent process needs to read its status. However, once the wait system call is complete, the zombie process is removed from the process table.

54) What is Network bonding in Linux?

Network bonding, also known as link aggregation, is a technique used to combine multiple network interfaces into a single interface. This process improves network redundancy and performance by increasing overall bandwidth and throughput.

55) What are the types of network bonding?

The different ways in which network bonding can be implemented are referred to as network bonding modes.

  • Mode-0 (balance-rr): Mode-0 is the default mode, and it works according to the round-robin policy. It provides features like load balancing and fault tolerance.
  • Mode-1 (active-backup): It works according to the active-backup policy. Only a single node works or responds during the failure of another node.
  • Mode-2 (balance-xor): This mode sets an exclusive-or (XOR) mode for offering fault tolerance and load balancing.
  • Mode-3 (broadcast): It works according to the broadcast policy. This mode configures a broadcast mode for offering fault tolerance and can be only used for specific goals.
  • Mode-4 (802.3ad): It is also called Dynamic Link Aggregation mode. It works according to the IEEE 802.3ad standard. It configures an IEEE 802.3ad dynamic link aggregation mode and makes aggregation groups that distribute similar duplex and speed settings.
  • Mode-5 (balance-tlb): It is also called Adaptive Transmit Load Balancing (TLB). It configures TLB mode for load balancing and fault tolerance.
  • Mode-6 (balance-alb): It is also called Adaptive Load Balancing. It configures ALB mode for load balancing and fault tolerance. It does not need unique switch support.
