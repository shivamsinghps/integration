Top 30 Most Asked Computer Science Interview Questions and Answers

1) What do you understand by a computer system?

A computer system is made up of interdependent parts, such as input devices, output devices, memory, CPU, and peripheral devices. These components are interconnected with the Operating System to manage and store data and information. Essentially, it serves as a tool for processing and storing information.

In today's world, computer systems are centered around digital processing devices. A computer system comprises of five primary hardware components which include Input, Output, Processing, Storage, and Communication devices.

2) What is a file?

A file is a fundamental element in the field of computer science. It is a designated space where data and information can be stored for future use. The data contained within a file can be retained permanently or until it is intentionally removed by the user. In order to identify a file, it is assigned a unique name and stored within a storage device.

3) What are the main components of a computer system?

There are three primary parts that make up a computer system.

  • The Central Processing Unit (CPU) is the primary component of a computer system that performs most of the processing. It is known as the brains of the computer and controls all the operations of the computer. It includes the control unit (CU) and arithmetic logic unit (ALU).
    CPU = CU + ALU
  • Memory: It includes memory like primary memory (RAM) and secondary memory (SSD).
  • Devices that are used to input or provide data to a computer system are known as input devices, while devices that display or convey information from the computer system are known as output devices. Examples of input devices include keyboards, mice, scanners, and cameras, while output devices include monitors, printers, and speakers. It includes input and output devices like keyboard, mouse, monitor, printer scanner, etc.

4) What do you understand by a class and a superclass?

In the world of Object-Oriented programming, the terms class and superclass are fundamental concepts. When working with Object-Oriented programming languages such as Java and C++, a class defines the attributes and behaviors of an object. It outlines the methods an object will use to respond to various messages.

A subclass is a class that has been created by inheriting from another class. It is also known as a derived class, extended class, or child class. The base class or parent class from which the subclass is derived is known as the superclass. The superclass serves as the foundation of the subclass being developed.

5) What is a processor in computer systems, and why is it used?

A processor is a vital component used to interpret and execute essential instructions required to operate a computer system. Its core functions include retrieving, deciphering, executing, and recording the outcome of each instruction. Often referred to as the brains of a computer, the processor plays a crucial role in the functioning of various devices such as laptops, smartphones, embedded systems, and more.

The processor is a computer component that consists of two essential parts, namely the Arithmetic Logic Unit (ALU) and the Control Unit (CU). The ALU is responsible for performing various mathematical operations, including addition, subtraction, multiplication, and division. In contrast, the CU acts as a traffic police by managing the execution of instructions. The processor also communicates with other hardware components of a computer system, such as input/output devices and memory/storage devices.

13) What are the different OOPs principles in Java?

The fundamental OOPs (Object-Oriented Programming System) principles are present in all Object-oriented programming languages including Java. Java programming language also adheres to these principles which are as follows:

  • Inheritance
  • Polymorphism
  • Abstraction
  • Encapsulation

In order to understand the core principles of OOPs, it's important to first grasp the basic concepts of Object-Oriented Programming. OOP is a programming paradigm that involves designing a program using classes and objects. Objects are entities that have a state and a behavior, and can either be physical or logical. Examples include chairs, tables, laptops, bikes, pens, pencils, and keyboards. Objects are essentially instances of a class and take up space in memory, complete with a unique address.

An example of an object is a cow, which possesses various attributes such as breed, color, and name, as well as performing various actions like producing milk and feeding.

In programming, an object is an instance of a class. It can be thought of as a real-world entity which has properties and behaviors. Objects take up physical space in a program as they are created in memory. They are used to perform operations, store data, and interact with other objects. A class, on the other hand, is a logical entity. It is a blueprint that defines the structure and behavior of objects of that class. It acts as a template for creating new objects with the same properties and methods. Classes do not take up any physical space in memory as they are not actual objects, but rather the definition of an object type.

Inheritance

In Java, Inheritance refers to the process by which a subclass acquires all the attributes and behaviors of a superclass. The superclass, also known as the parent class, passes on its properties and methods to the subclass, which is known as the child class. The primary purpose of inheritance is to enable runtime polymorphism and promote code reuse.

Polymorphism

Polymorphism is a crucial concept in object-oriented programming, allowing for tasks to be performed in multiple ways and data to be processed in various forms. Polymorphism is a combination of the words poly, meaning many, and morphism, meaning types. In Java, it is possible to achieve polymorphism through method overloading and method overriding.

Abstraction

Abstraction refers to the technique of concealing intricate details while presenting only its crucial functionalities. It omits extraneous information and presents a simplified version of the subject matter to the user. An excellent example is a phone call where the user is unaware of the underlying technical process involved in making the call.

Encapsulation

Encapsulation is a technique used to combine data into one unit. It functions as a tool that binds code and the data it is manipulating. Encapsulation acts as a safeguard that does not allow external code to access protected information. An example of encapsulation is seen in Java classes.

15) What is multithreading in an Operating System?

Multithreading refers to the capability of a program or an operating system process to handle multiple users and requests simultaneously, without requiring multiple copies of the code to be running on the system. Essentially, a CPU can execute multiple threads at once, allowing for greater efficiency and improved performance. It should be noted that multithreading is distinct from multiprocessing, which involves running multiple processes on a system at the same time.

17) What is the key difference between Abstract class and Interface?

The following outlines the primary distinctions between Abstract classes and Interfaces:

Abstract class Interface
An Abstract class does not allow for any kind of inheritance. The Interface has the ability to inherit from multiple sources simultaneously.
An abstract class can include both abstract and non-abstract methods. An Interface in Java is a type which can contain only abstract methods. However, newer versions of Java (starting with Java 8) have introduced the ability for Interfaces to also include default and static methods.
An abstract class is capable of containing both static and non-static variables, as well as both final and non-final variables. It is possible for an Interface to contain exclusively static and final variables.
One characteristic of an abstract class is that it may include a constructor. The Interface lacks a constructor.
An Abstract class has the ability to furnish an implementation of an Interface. An interface is incapable of supplying the concrete implementation of an abstract class.
You can declare an abstract class by using the keyword "Abstract." To declare an interface in programming, the keyword "interface" is used.
It is possible for an Abstract class to inherit from a Java class and also implement multiple Java interfaces at the same time. It is possible for a Java interface to inherit from another Java interface, but it cannot extend from a class.
An abstract class can be assigned any access modifier without any issue. The access modifier for an interface in programming must be public.
You can inherit an abstract class by using the keyword "extends". The implementation of an Interface is achieved through the utilization of the keyword "implements".
Java's Abstract class allows for the inclusion of class members with different access levels, such as private or protected. The access modifier of members defined in a Java interface is public by default.

19) What are the key differences between C and C++ programming languages?

Here are some significant contrasts between the C and C++ programming languages:

C C++
C is classified as a Procedure-Oriented or Procedural Programming language, which means it lacks support for several fundamental concepts found in Object-Oriented languages. These include encapsulation, inheritance, abstraction, and polymorphism. C++ is a programming language that has been designed with the ability to support Object-Oriented Programming. This means that it has features such as inheritance, encapsulation, polymorphism, and abstraction.
Dennis Ritchie developed the C language between 1969 and 1973 at AT&T Bells Laboratory. In 1979, Bjarne Stroustrup created the C++ programming language.
C++ language encompasses a larger scope than the C language, which is a subset of it. C++ is a programming language that includes all of the features of the C language and expands upon them.
The programming approach used in C language is Top-Down, meaning that the program design starts with the main function and is gradually broken down into smaller modules. The programming technique employed in C++ is the bottom-up approach, which involves breaking down complex problems into smaller, more manageable ones and gradually building up solutions from these smaller components.
One way to organize a large C programming code is by dividing it into smaller sections known as functions. These functions are individual units of code that perform specific tasks and simplify the overall program structure. C++ is a programming language that allows for the organization of large programs into smaller, more manageable parts called objects and classes. By breaking down the code into these components, the entire program can be easier to understand and modify.
The file extension used in the C programming language is .c. The file extension used in C++ programming language is ".cpp".
C is a programming language that is procedural in nature, used for coding development. It is often referred to as a mid-level language due to the combination of high and low-level features. C++ is considered a hybrid language due to its ability to support two primary programming paradigms procedural and object-oriented programming.
The C programming language consists of 32 keywords. The C++ programming language has a total of 63 keywords.
In C programming language, there is no built-in support for the concept of information hiding. C++ uses Encapsulation to hide data, which helps to ensure that data structures and operators are utilized correctly.
The C programming language only allows the use of data types that are built-in. C++ has the capacity to accommodate both pre-existing and custom-made data types.
In the C programming language, there is a clear distinction between data and functions due to its procedural nature. C++ is a programming language that follows the object-oriented paradigm, where data and functions are combined into an object.
C is characterized as both a procedural programming language, which means it follows a step-by-step approach to solve problems, and a function-driven language, where developers define functions to perform specific tasks. C++ is categorized as an object-oriented programming language, which makes it an object-driven language as well.
It is not possible to do function and operator overloading in C language. In C++, it is possible to overload functions and operators, which allows the programmer to define multiple functions or operators with the same name but different parameters or behaviors.
C language is a programming language that relies heavily on functions to perform tasks. C++ is a programming language that is heavily focused on object-oriented programming.
It is not possible to define functions within structures in the C programming language. C++ allows for the use of functions within structures.
The Namespace feature is not available in the C programming language. The C++ programming language utilizes the Namespace concept to prevent naming conflicts.
The standard header file utilized in the C programming language is called stdio.h. In the C++ programming language, the header file utilized is known as iostream.h.
C does not support Reference variables. C++ supports Reference variables.
In the programming language C, there is no provision for implementing virtual and friend functions. In C++, there are two types of functions - virtual and friend functions.
The key emphasis of the C programming language lies in its methodology rather than the data itself. The emphasis of C++ programming language is on data rather than methods or procedures.
C programming language provides functions for allocating and deallocating memory dynamically, namely malloc() and calloc(), and free(). These functions allow programs to acquire memory at runtime as needed, and then release it when it is no longer in use. The memory allocation process in C++ involves the use of a specific operator, while memory de-allocation is carried out using another operator specifically designed for that purpose.
C programming language does not have built-in support for handling exceptions. Exception handling is a feature supported by the C++ programming language.
The C programming language uses scanf() and printf() functions for input and output operations. When using the C++ language, you can utilize the cin and cout functions for input and output purposes. These functions are used to prompt users for input and output results, respectively.
In C, access modifiers are not supported for structures. In C++, access modifiers are associated with structures.
In a C structure, it is not possible to declare a function within the structure definition. In C++, it is possible to define a function as part of a structure, allowing it to be treated as a member function.

21) What are the key differences between compiler and interpreter?

Compiler Interpreter
A compiler reviews the entire program in one go. An Interpreter processes a program one statement at a time by translating it into a machine-readable format.
When compiling code, the entire code is scanned at once and any errors are displayed at the end as a group. The process of reading a program's code by the interpreter is done line by line, thus making it possible to identify errors on each line accordingly.
The programming language has a high execution speed, and the resultant compiled code performs faster than other alternatives. The interpreter executes code line by line, which makes it less efficient compared to compiled languages. The interpreted code also tends to run slower than compiled code.
Creating a compiler is a challenging task since it cannot anticipate every possible outcome. Interpreters are most suitable for programming and software development.
The advantage of using compilers is that they have faster execution times compared to other methods. Interpreters have a lower preference as they tend to execute object code at a slower pace than other alternatives.
The process of converting high-level source code into executable object code is done by compilers. Interpreters don't transform source code to object code. They examine it line by line.
A compiler doesn't need the original source code to be kept for future use. In order for interpreters to run programs, they need access to the original source code.
The compiled program can run separately without the need for the compiler to be active in memory. During the process of interpretation, the interpreters are present in the memory.
There are several programming languages that utilize compilers, such as C, C++, and C#. There are several programming languages that utilize interpreters instead of compilers, some of which include Python, Ruby, Perl, SNOBOL, and MATLAB.

28) What are the key differences between primary and secondary memory?

Computer memory can be categorized into two main types - primary memory and secondary memory. Primary memory is the computer's internal memory that is directly accessed by the CPU for storing and retrieving data. In contrast, secondary memory refers to an external storage device that can be used for permanent data storage. While both types of memory serve the same purpose, they function differently. Here are some key differences between the two types of computer memory.

Primary memory Secondary memory
Main memory, also known as primary memory, is a computer's primary storage area that can be directly accessed by the Central Processing Unit (CPU). The reason why secondary memory must be transferred to the primary memory before being accessed by the processor is because the processor has no direct means of interacting with the secondary memory.
The memory that stores data temporarily until the execution of a process is known as primary memory. Secondary memory refers to an external storage device that is utilized for permanent storage of data or information.
Primary memory exhibits different properties depending on its type. Volatility characterizes the nature of RAM, while Non-volatility characterizes that of ROM. Secondary storage is a type of computer memory that retains the stored data even when the power is turned off and is commonly referred to as non-volatile memory.
Semiconductor-based memory components are utilized as primary memory in various electronic devices. Secondary memory devices rely on magnetic or optical technologies for storage. These types of memory allow for the long-term preservation of information in computing systems.
Compared to secondary storage devices, primary memory devices have a higher price point. The cost of secondary memory devices is relatively lower compared to primary memory devices.
Primary memory can be referred to as Main memory or Internal memory. Secondary memory can also be referred to as auxiliary memory or external memory.
Primary memory consists of various types of memory devices such as RAM, ROM, Cache memory, PROM, EPROM, and Registers. They are the primary sources of memory in a computer system. Secondary memory refers to computer storage that is non-volatile, meaning that it retains its contents even after the computer is turned off. Examples include hard disks, floppy disks, and magnetic tapes.
