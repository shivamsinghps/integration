Tableau Interview Questions and Answers

Below is a compilation of frequently asked Tableau interview questions and their corresponding answers.

1) What is Tableau?

  • A tableau is a popular and highly effective tool for visually presenting data in the field of business intelligence.
  • The data is transformed into a format that is simple and easy to comprehend.
  • It offers a platform that allows users to create visually appealing and interactive dashboards that can be easily shared with others.
  • Tableau does not necessitate any specialized technical or programming abilities.

2) Explain the features of Tableau?

Tableau has three key functionalities that differentiate it from other data visualization tools.

  1. Data Blending: Data blending is an essential feature in Tableau. It is used when we combine related data from multiple data sources, which you want to analyze together in a single view, and represent in the form of a graph.
  2. Real-time Analysis: Real-Time Analysis makes users able to quickly understand and analyze dynamic data when the Velocity is high, and real-time analysis of data is complicated. Tableau can help extract valuable information from fast-moving data with interactive analytics.
  3. Collaboration of data refers to the process of working together to share and analyze information. It involves multiple parties coming together to pool their data resources, knowledge, and insights to gain a better understanding of a particular topic. This approach allows for a more comprehensive and holistic interpretation of data, leading to better decision-making and problem-solving. Data analysis is not isolating task. That's why Tableau is built for collaboration. Team members can share data, make follow up queries, and forward easy-to-digest visualizations to others who could gain value from the data. Making sure everyone understands the data and can make informed decisions is critical to success.

3) How many tools in Tableau?

Tableau consists of five distinct tools.

  1. Tableau Desktop: Tableau Desktop establish connectivity between the Data Warehouse and other various types of files. The dashboards and the workbooks created here can be either shared locally or publicly. It allows us to code and customizes reports. Right from establishing the stories, charts to blending them all to form a dashboard, all the necessary work is created in Tableau Desktop.
  2. Tableau Public: This Tableau version is specially built for cost-effective users. The word 'Public' means that the created workbooks cannot be saved locally. They should be kept on the Tableau's public cloud, which can be accessed and viewed by anyone.
  3. Tableau Online: Its functionality is similar to the Tableau server, but data is stored on the server that hosted on the cloud, which is maintained by the Tableau group. There is no storage limit on the data which is published in the Tableau Online. Tableau Online creates a direct link over 40 data sources who are hosted in the cloud such as the The following are some examples of databases that are commonly used in the industry: Hive, MySQL, Spark SQL, and Amazon Aurora. These databases are known for their efficiency and reliability in handling large amounts of data., and many more.
  4. Tableau Server: The software is correctly used to share the workbooks, visualizations, which is created in the Tableau Desktop application over the organization. To share dashboards in the Tableau Server, you should first publish your workbook in the Tableau Desktop. Once the workbook has been uploaded to the server, it will be accessible only to the authorized users.
  5. Tableau Reader: Tableau Reader is a free tool which allows us to view the visualizations and workbooks, which is created using Tableau Desktop or Tableau Public. The data can be filtered, but modifications and editing are restricted. There is no security in Tableau Reader as anyone can view workbook using Tableau Reader.

4) Explain all data terminologies in Tableau?

Tableau has different types of data terminologies. These include:

  1. Bookmark: A .tbm document in the bookmarks folder in the Tableau repository that contains a single worksheet. It helps in improving data analysis. Unlike, web browser bookmarks, .tbm files are a compatible way to display various studies quickly.
  2. Workbook: A workbook is a file with .twb extension that holds one or more worksheets as well as dashboards and stories.
  3. Dashboard: The dashboard is a combination of several views that are arranged on a single page. In Tableau, dashboards are used to observe and compare a variety of data together, and also it allows interacting with other worksheets.
  4. Data Source Page: Data Source is a page where you can set up your data source. Does this data source page generally consist of four main areas? Join area, left pane, a preview area, and metadata area.
  5. Worksheet: The worksheet is a collection of sheets. It's a place where you build views of your data by dragging various fields onto the shelves.
  6. Dimensions: Dimension is commonly known as a field of categorical data. Dimensions hold discrete data such as members and hierarchies that cannot be aggregated. It also contains characteristic values such as dates, names, and geographical data. The dimensions used to reveal details of your information.
  7. Measures: measures are the measurable quantities of the data, which can be analyzed by a dimension table. Measures are stored in a table which contains foreign keys referring uniquely to the associated dimension tables. The table supports data storage at the atomic level and thus, allows the number of records to be inserted at one time.
    For example, a Sales table can have a product key, customer key, promotion key, items sold, referring to a specific event.
  8. Filters shelf: Filter shelf is located on the left side of the workbook. Filters shelf is used to exclude the data from a view by filtering it using both dimensions and measures.
  9. Pages shelf: Page shelf is on the left side of the view. With the help of the page shelf, you can split a view into a sequence of pages based on the values and members in a continuous or discrete field. Adding a field with the pages shelf is similar to adding a field in rows shelf. For each new row, a new page is created.
  10. Marks card: Marks card is on the left side of the worksheet. The user can drag fields to the control mark properties such as color, type, shape, size, label, detail, and tooltip.

5) What is the difference between .twb and .twbx extension?

.twb .twbx
A .twb file is an xml-based file that stores the data related to your dashboards, sheets and stories in Tableau. To share a workbook in Tableau, you need to send both the workbook file and the data source file as it does not contain any data.
A .twbx is a collection of files that have been merged together into a compressed package. A workbook package contains all the necessary files required to produce the Excel document, including the data source file and any other accompanying files such as images.

Tableau software automatically assigns data types and expresses fields based on the data source. If the data source has a defined data type, Tableau will use it. However, if the data source doesn't have an assigned data type, Tableau will assign one. Tableau offers various data types, which include:

  • Date values
  • Text values
  • Numerical values
  • Rewritten: An area of data that deals with the recording and management of dates and times. It is used to represent specific moments in time, such as when an event occurred or when a transaction was made. This type of data is important for many applications and can be stored in various formats, such as numeric values or text strings. Proper management of date and time data is essential to ensure accuracy and consistency in the organization and analysis of data.
  • Reworded: Boolean values, which refer to conditions that are either true or false, are an important concept in programming.

8) What are the different types of joins in Tableau?

Tableau's join feature shares similarities with SQL joins and encompasses five primary types of joins.

A cross join is a type of join in which every row in one table is paired with every row in another table to produce a new, larger table. This results in all possible combinations of the rows from both tables.

An inner join is a type of SQL join that returns only the rows that match in both tables being merged. It filters out the unmatched rows, leaving only the matching ones.

The left outer join is a type of join in which matching rows from the joined tables are returned, along with any non-matching rows from the left table. In the result, the attributes from the right table are populated with NULL values for the non-matching rows.

A right outer join is a type of merging operation between two tables. It returns both matching rows and non-matching rows from the right table, placing NULL values in the attributes originating from the left table for non-matching rows.

The full outer join is a useful concept in database management that allows tables to be combined. It includes all data from both tables, making it a valuable tool in merging information.

9) How many tables you can join in Tableau?

Tableau allows for the joining of up to 32 tables.

10) Explain the different connections you can make with your datasets?

There are two methods to link the datasets together.

  1. Live: Live connection sends queries to your database and retrieves data. These queries will return whatever data is currently in the database.
  2. Extract: Extracts connection saved subsets of data that use to improve performance or to take the advantages of Tableau functionality which are not supported or available in your original data.

11) What are Shelves?

Shelves are essential components of a worksheet's user interface, positioned on the top and left-hand sides of the view. By selecting and dragging fields onto the appropriate shelves, users can create bespoke visualizations. Specific mark types may have additional shelves available for selection.

12) What are the Sets?

Tableau offers a feature called "sets" which allow users to create subsets of their data based on specific conditions defined by the user. These sets can only be created using the dimension field.

Tableau recognizes two types of sets:

  1. Dynamic Sets: The values or members in the dynamic sets get change when the underlying data changes.
  2. Fixed Sets: The values or members in the fixed sets does not change when the underlying data changes.

13) What are Groups?

A group refers to a collective of individuals that form broader categories within a given dimension.

14) What is the Hierarchical Field?

Tableau employs the use of hierarchical fields in order to allow users to analyze and study their data in a more detailed and specific manner. By drilling down the data, users are able to view the data at a more granular level.

15) What is a Tableau Data Server?

Tableau server plays an intermediary role between data and Tableau users. With Tableau Data Server, you can upload and share data extracts using preserved database connections while also reusing field metadata and calculations.

The ability to save changes made to a calculated field, dataset, aliases, definitions, and parameters can enhance collaboration and standardization, as these changes can be shared with others and managed centrally. This can provide more secure data management and utilisation of the server's resources, as queries can be run on extracts without the need to transfer data to local devices.

16) What is the Tableau Data Engine?

Tableau Data Engine is an essential component of Tableau software that operates as an analytical database with the ability to deliver quick query results. It has predictive performance and effortlessly integrates with existing data infrastructures. Furthermore, Tableau Data Engine saves time as it does not require the loading of entire datasets into the memory.

When dealing with a vast amount of data, the initial phases of creating indexes, importing and sorting data can be time-consuming. However, once completed, operations speed up considerably. The Tableau Data Engine doesn't rely on in-memory technology, meaning that data is stored on disk rather than in RAM, which is minimally utilized.

18) Define Dual axis.

A dual-axis chart is a graphical representation of two measures using two different chart types. To create this chart, a date column and two measures are required.

In the graph, distinct scales are implemented to aid in comprehending the measurements presented.

19) What is the difference between the Tree map and Heat map?

Tree map Heat map
A treemap is capable of visualizing hierarchical data and displaying part-to-whole relationships similarly to a sunburst chart. A heat map can compare categories with color and size
In the heat map, you can compare two different measures together.

20) What is the difference between Data Joining and Data blending?

Data joining Data blending
Data joining refers to the process of merging data from a common source. This method is used to combine information from different data sources for analysis purposes. To incorporate two distinct data sources in a report, it is necessary to perform data blending.

21) What is TDE file?

A TDE file is a Tableau desktop file that has a file extension of .tde. It is used to store data that has been extracted from external sources such as CSV files, Microsoft Excel, and MS Access.

TDE files possess two qualities that make them suitable for facilitating analytics and data exploration.

  • TDE is a database system that stores data in a column-wise format.
  • TDE efficiently utilizes different types of computer memory such as RAM and hard disk to achieve the best performance possible according to each component's specific properties.

22) What is the Story in Tableau?

A story can be comprised of a collection of slides or worksheets that work cohesively to convey a message or information.

One way to effectively convey information is by using storytelling techniques. By creating stories, information can be presented in a way that demonstrates how various facts are related, provides context, and shows how decisions can impact outcomes. Each segment of a story is called a story point.

23) What is the difference between discrete and continuous data in Tableau?

Discrete Continuous
Discrete data refers to values that are considered as separate and distinct entities, rather than a continuous or connected set of values. Continuous data is utilized for the purpose of measuring data that is continuous.
The variable referred to in the statement has limited options for values and can only take on individual values within a specified range. The variable has the ability to assume a wide range of values, be it finite or infinite.

24) How many numbers of rows Tableau can utilize at one time?

Tableau has the ability to handle large sets of data as it is not limited by the number of rows present in a table. This software efficiently accesses and manages petabytes of data by retrieving only the necessary rows and columns required.

25) Can Tableau be installed on macOS?

Tableau Desktop is compatible with both Mac and Windows operating systems, allowing users to install the software on either type of device.

26) Tableau software is suitable for strategic acquisition or not?

Tableau software is a valuable tool for strategic acquisition as it allows for deeper insights into data compared to other tools.

27) Can we place an excel file in a shared location in Tableau?

Certainly, we have the option of storing an Excel file in a shared location, but to ensure optimal performance, it's advisable to utilize extract functionality instead.

28) Suppose license expires today, will users be able to view dashboard or worksheet which published on the server?

If your Tableau Desktop license runs out today, accessing any dashboards or worksheets will not be possible. This is due to the fact that the username associated with the server will be marked as unlicensed, therefore denying access.

It is possible for some individuals to be unable to access a particular website due to ownership changes made by the site administrator. Ownership transfers may be necessary to prevent extraction failure for other users.

29) What are the different Tableau files?

  • Bookmarks: It contains only single worksheet and its easy way to share your work.
  • Workbooks: Workbook can hold one or more dashboards and worksheets.
  • Packaged workbooks: It contains the workbook along with any supporting local file data and background images.
  • Data extraction files: Data extraction files are a local copy of a data source or a subset.
  • Data connection files: Data connection file is a small XML file that contain various connection information.

30) What is the difference between the published data source and an embedded data source?

Published data source Embedded data source
The data source file (DSN) is a file that provides connectivity information for a database or other data source. This information is not specific to any particular workbook and can be referenced by multiple workbooks. The file includes details for establishing connections and is linked to a specific workbook.
