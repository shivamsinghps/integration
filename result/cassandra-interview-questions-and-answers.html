Cassandra Interview Questions

Here are some common questions that come up in Cassandra interviews along with their answers. The questions are frequently asked and are meant to help prepare for an interview.

1) Explain Cassandra.

Cassandra is a widely used NOSQL database management system that is designed to handle massive datasets. It is a distributed and open-source database that ensures high availability with no risk of failure.

2) In which language Cassandra is written?

Cassandra was developed in Java and is known for its ability to handle enormous amounts of data. Its flexible schema allows for easy adaptation to changing data needs. Originally created by Facebook, it is extremely scalable and ideal for managing big data.

3) Who was the original author of Cassandra?

Avinash Lakshman and Prashant Malik are the creators of Cassandra. Originally developed at Facebook, the primary purpose was to support the Facebook inbox search function.

4) Which query language is used in Cassandra database?

Cassandra has developed its own query language called Cassandra Query Language (CQL), which provides an uncomplicated way of accessing Cassandra. CQL is a substitute for the standard SQL language.

5) What are the benefits/advantages of Cassandra?

  • Cassandra is a technology that provides fast and up-to-date performance, streamlining the tasks of various professionals including Developers, Administrators, Data Analysts, and Software Engineers.
  • The technology offers the advantage of being highly scalable with flexibility to easily adjust to varying levels of demand. It can be expanded or reduced as needed to accommodate changing requirements.
  • Data can be copied and stored on multiple nodes to ensure that there is redundancy in case of any failures or errors.
  • A distributed management system offers a high degree of reliability as there is no single point of failure.
  • In a cluster, each individual node has access to distinct data and is capable of handling various types of requests.

6) Where Cassandra stores its data?

Cassandra's data is stored in a repository called the data dictionary.

7) What was the design goal of Cassandra?

Cassandra was developed with the primary objective of enabling distributed processing of massive amounts of data across multiple nodes in a fault-tolerant manner, without any single point of failure.

8) How many types of NoSQL databases? Give some examples.

NoSQL databases can generally be classified into 4 main types.

  • There are two main types of document stores, namely MongoDB and CouchDB. These stores are designed to store and manage unstructured data.
  • Two popular types of key-value stores are Redis and Voldemort. These database systems are designed to efficiently store and retrieve data based on keys and provide high performance and scalability. Redis and Voldemort are commonly used in applications that require fast and reliable access to data, such as e-commerce platforms and social media websites.

11) What is keyspace in Cassandra?

A keyspace in Cassandra serves as a way to define the data replication across nodes. Each node in a cluster contains one keyspace. It essentially is a namespace that identifies how data is distributed and stored within the database.

12) What are the different composite keys in Cassandra?

Composite keys are employed in Cassandra database to identify a key or column by combining various data types. There are two categories of composite keys that can be used in Cassandra.

  • Row Key
  • Column Name

13) What is data replication in Cassandra?

Data replication refers to the process of duplicating data from one computer or server's database to another to ensure that all users can access the same information. In Cassandra, data replication involves storing replicas on multiple nodes in order to guarantee reliability and resilience in the face of faults. The decision of which nodes to place the replicas on is governed by the replication strategy chosen.

14) What is node in Cassandra?

A node is a location within Cassandra where data is stored.

A data center is a collection of interconnected clusters that store and manage large amounts of data.

16) What do you mean by commit log in Cassandra?

Cassandra utilizes a mechanism called commit log for crash-recovery purposes. This involves writing every write operation to the commit log.

17) 17) What do you mean by column family in Cassandra?

A column family refers to a structured format used in relational databases where data is organized in rows and columns. It is essentially a table consisting of a series of ordered rows.

18) What do you mean by consistency in Cassandra?

The concept of consistency in Cassandra focuses on how to keep a synchronized and updated version of a row of data in Cassandra and its corresponding replicas.

20) What is tunable consistency in Cassandra?

Cassandra stands out from other database management systems due to its remarkable feature, Tunable Consistency, which makes it a popular choice among users. This feature ensures that the data rows on all replicas are always synchronized and up-to-date, a crucial aspect of consistency. Additionally, Cassandra's users can select the best consistency level for their use cases, making it a versatile tool that caters to users' specific needs.

23) How does Cassandra perform write function?

In Cassandra, writing data is done through the use of two commits that are applied to the database.

  • In the process of committing data to a database, the first commit is first applied to disk while the second commit is stored temporarily in a memory structure referred to as memtable.
  • After successfully applying both commits, the write operation is completed.
  • The data is stored in a table structure using SSTables which stands for sorted string tables.

24) What is memtable?

A memtable is a type of cache space that holds data in key and column format within a system's memory. This in-memory cache is sorted by key, with each ColumnFamily having its own distinct memtable for fetching column data using keys. The memtable temporarily stores incoming writes until it reaches its limit, at which point it is flushed out.

25) What is SSTable?

An SSTable is a critical file in Cassandra that stores sorted string tables containing data from written memtables. These files are unique for each Cassandra table and are stored on disk.

26) How the SSTable is different from other relational tables?

When data is written into an SSTable in Cassandra, it cannot be modified or deleted later on. Each SSTable is accompanied by three distinct files that include a partition index, partition summary, and a bloom filter.

27) What are the management tools in Cassandra?

DataStaxOpsCenter is an online monitoring and management tool designed specifically for DataStax and Cassandra cluster. It's available for free download and comes with an additional OpsCenter edition.

SPM is a monitoring tool that mainly deals with monitoring Cassandra metrics and different OS and JVM metrics. It has the capability to monitor numerous Big Data platforms including Hadoop, Solr, Spark, Storm, and Zookeeper, besides Cassandra.

29) What is cluster in Cassandra?

Cassandra's cluster is responsible for organizing the nodes in a ring structure and distributing data to them. The nodes have replicas that serve as backups in case of data handling issues, and keyspaces are encompassed within the cluster.

30) What is the role of ALTER KEYSPACE?

The ALTER KEYSPACE command is utilized to modify the settings of DURABLE_WRITES along with its associated properties.

31) What do you mean by Cassandra-Cqlsh?

Cqlsh is a tool that enables executing CQL commands and is used as a shell for the Cassandra query language. It provides a user-friendly interface for interacting with Cassandra databases.

32) What are the differences between a node, a cluster, and datacenter in Cassandra?

A node refers to a singular computing machine that is responsible for running Cassandra.

A cluster refers to a group of nodes that are gathered together based on similar data types.

A datacenter is a crucial element in providing services to customers across different locations. It allows for the classification of various nodes within a cluster into separate data centers.

33) What is the use of Cassandra CQL collection?

A collection in Cassandra CQL is a mechanism for gathering and storing related data in a column. Each collection contains data of the same type. CQL supports three different types of collections.

  • SET: It is a collection of unordered list of unique elements.
  • List: It is a collection of elements arranged in an order and can contain duplicate values.
  • MAP: It is a collection of unique elements in a form of key-value pair.

34) What is the use of Bloom Filter in Cassandra?

When a data request is made, the Bloom filter first checks if the requested data is present in the row of SSTable before initiating any disk I/O actions.

35) How does Cassandra delete data?

To delete a row in Cassandra, one must assign the value of a column to a special value called "Tombstone."

36) What is SuperColumn in Cassandra?

SuperColumn is a distinctive component in Cassandra that comprises a grouping of similar data collections. It consists of key-value pairs, where the values are columns.

37) What is the difference between Column and SuperColumn?

"Column" and "SuperColumn" are two terms that are used in the context of NoSQL databases. These terms have distinct meanings, and it is important to understand their differences. A "Column" refers to a unit of data in a NoSQL database. It consists of a name, a value, and a timestamp. This data is stored in a row of a column family. On the other hand, a "SuperColumn" is a collection of Columns within a NoSQL database. It is essentially a group of related Columns that are stored together, and it can be useful for organizing data in a hierarchical manner. Therefore, it is essential to understand the difference between these two terms to effectively utilize a NoSQL database.

  • The columns in a database are stored as strings, while the SuperColumn contains a Map of Columns with varying data types.
  • Super Columns have a different structure compared to Columns. In Super Columns, the timestamp is not included as the third component.

38) What is Hadoop, HBase, Hive and Cassandra? Specify similarities and differences among them.

Hadoop, HBase, Hive, and Cassandra are software products developed by the Apache Software Foundation.

Apache Hadoop offers extensive support for file storage and grid compute processing through its MapReduce system. Additionally, it provides Apache Hive, which serves as a SQL-like interface on top of Hadoop, making data processing more user-friendly. Another column family storage system, similar to Google's Big Table, is Apache HBase, while Apache Cassandra works on the same principles, but also incorporates a Dynamo topology and consistency.

The close() method in Cassandra is used to terminate the current session instance.

40) Which command is used to start the cqlsh prompt?

To launch the cqlsh prompt, one can make use of the "cqlsh" command.

41) What is the usage of "cqlsh-version" command?

You can determine the version of cqlsh you are currently using by running the command "cqlsh-version".

43) What is Kundera in Cassandra?

Kundera is a Java-based implementation of object-relational mapping (ORM) in Cassandra. It uses Java annotations to achieve this mapping.

44) What do you mean by Thrift in Cassandra?

Thrift is a client application programming interface used for the purpose of connecting and communicating with the Cassandra Server in remote procedure call (RPC) architecture.

45) What is Hector in Cassandra?

Hector is a historic Cassandra customer that dates back to the early days. It is a freely accessible Java-based open source platform with a MIT permission.
